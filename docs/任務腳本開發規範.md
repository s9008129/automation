# 任務腳本開發規範

> 本文件說明如何撰寫符合本專案架構的自動化任務腳本。
> 無論是手動撰寫或請 AI 生成，都必須遵守以下規範。

---

## 快速開始

複製範本並修改：

```powershell
# 複製範本（Windows 指令）
Copy-Item src\templates\task-template.ts src\qiz-批次簽核.ts
```

然後用文字編輯器或 AI 根據實際需求修改。

---

## 腳本命名與放置規範

**所有任務腳本必須放在 `src\` 目錄下**，禁止放在根目錄。

命名格式：

```
src\{系統代碼}-{任務名稱}.ts

系統代碼範例：
  qiz   → QIZ 電子表單系統
  eip   → EIP 財稅雲端平台入口
  vst   → VST 資料系統
  
範例：
  src\qiz-批次簽核.ts
  src\eip-公文簽核.ts
  src\vst-資料匯出.ts
```

## 必要技術規範

### 1. 模組系統：使用 ESM import，禁止 require

```typescript
// ✅ 正確
import { chromium } from 'playwright';
import * as fs from 'fs';

// ❌ 禁止
const { chromium } = require('playwright');
```

### 2. 環境變數：使用 loadDotEnv()，禁止 dotenv 套件

```typescript
// ✅ 正確：使用專案提供的 helper
import { loadDotEnv, requireEnv } from './lib/env.js';
loadDotEnv();
const password = requireEnv('PASSWORD');

// ❌ 禁止：不得引入 dotenv
import dotenv from 'dotenv';  // 不允許
dotenv.config();              // 不允許
```

### 3. 瀏覽器：使用 channel: 'msedge'，headless: false

```typescript
// ✅ 正確：使用系統已安裝的 Edge，不需要下載瀏覽器
const browser = await chromium.launch({
    channel: 'msedge',
    headless: false,    // 保留畫面，讓使用者觀察
});

// ❌ 禁止
const browser = await chromium.launch({ headless: true });     // 不允許隱藏視窗
const browser = await chromium.launch({ channel: 'chrome' }); // 請用 msedge
```

### 4. 敏感資訊：從 .env 讀取，禁止硬編碼

```typescript
// ✅ 正確
const username = requireEnv('USERNAME');
const password = requireEnv('PASSWORD');

// ❌ 禁止：硬編碼密碼
const password = 'my-secret-password';  // 絕對不行
```

### 5. CAS SSO 登入：等待 redirect，禁止硬編碼 ticket

```typescript
// ✅ 正確：等待 SSO redirect 完成
await page.getByRole('button', { name: '憑證登入' }).click();
await page.waitForURL('**/eip100/eip100m_v1.jsp**', { timeout: 30000 });

// ❌ 禁止：硬編碼 ticket（每次都不同）
await page.goto('...?ticket=ST-14549-...');  // 絕對不行
```

### 6. 分頁清單：必須用迴圈處理所有頁面

```typescript
// ✅ 正確：用迴圈處理所有頁面
let pageNum = 1;
while (true) {
    log.info(`處理第 ${pageNum} 頁…`);
    
    // 執行本頁操作
    // ...
    
    // 嘗試前往下一頁
    const nextBtn = page.getByRole('button', { name: '下一頁' });
    if (!(await nextBtn.isVisible()) || await nextBtn.isDisabled()) {
        log.info('已處理完所有頁面');
        break;
    }
    await nextBtn.click();
    await page.waitForLoadState('networkidle');
    pageNum++;
}

// ❌ 禁止：只處理第一頁
const items = await page.getByRole('row').all();  // 漏掉後續頁面
```

### 7. 使用者訊息：使用正體中文

```typescript
// ✅ 正確
log.info('開始執行批次簽核…');
log.error('找不到簽核按鈕，請確認頁面已正確載入');
throw new Error('登入失敗，請確認帳號密碼是否正確');

// ❌ 禁止：英文使用者訊息
log.info('Starting batch signing...');
throw new Error('Login failed');
```

## 建議使用的共用 Helper

本專案提供以下共用模組，建議在任務腳本中使用：

### env.ts — 環境變數載入

```typescript
import { loadDotEnv, requireEnv, getEnv } from './lib/env.js';

loadDotEnv();                              // 載入 .env
const user = requireEnv('USERNAME');       // 必要設定，缺少時報錯
const port = getEnv('PORT', '8080');       // 可選設定，有預設值
```

### logger.ts — 統一日誌

```typescript
import { createLogger } from './lib/logger.js';

const log = createLogger('批次簽核');
log.info('開始執行');
log.success('簽核成功');
log.warn('發現空白項目，略過');
log.error('網路連線中斷');
```

### error.ts — 錯誤處理

```typescript
import { handleTaskError, formatError } from './lib/error.js';

// 在 main() 的 catch 中使用
main().catch(err => handleTaskError(err, '批次簽核'));

// 格式化錯誤用於日誌
const fmt = formatError(err);
log.error(fmt.message, fmt.stack);
```

### browser.ts — 瀏覽器操作

```typescript
import { launchEdge, getNestedIframe } from './lib/browser.js';

const browser = await launchEdge();
// 或使用 QIZ 系統的雙層 iframe
const v1Frame = getNestedIframe(page, 'mainFrame', 'v1');
await v1Frame.getByRole('button', { name: '我的批次簽核' }).click();
```

### task-context.ts — 任務上下文

```typescript
import { createTaskContext, ensureDir } from './lib/task-context.js';

const ctx = createTaskContext('批次簽核');
ensureDir(ctx.outputDir);  // 建立輸出目錄
if (ctx.isDryRun) {
    log.warn('Dry-run 模式：不實際執行');
}
```

## 標準腳本結構

每個任務腳本應遵循以下結構：

```typescript
/**
 * 任務說明
 * 執行方式：.\run-task.ps1 src\qiz-批次簽核.ts
 */

// 1. ESM import（按需引入）
import { chromium } from 'playwright';
import { loadDotEnv, requireEnv } from './lib/env.js';
import { createLogger } from './lib/logger.js';
import { handleTaskError } from './lib/error.js';
import { createTaskContext, ensureDir } from './lib/task-context.js';

// 2. 任務常數
const TASK_NAME = '批次簽核';
const TARGET_URL = 'https://your-system.example.com/';

// 3. 主要邏輯（async function）
async function main(): Promise<void> {
    loadDotEnv();
    const ctx = createTaskContext(TASK_NAME);
    const log = createLogger(TASK_NAME);
    
    // ... 任務邏輯 ...
}

// 4. 進入點（固定格式）
main().catch(err => handleTaskError(err, TASK_NAME));
```

## 常見問題

### Q：腳本執行時找不到 Edge，怎麼辦？

確認系統已安裝 Microsoft Edge（系統內建的版本即可）。
`channel: 'msedge'` 使用系統安裝的 Edge，不需要 Playwright 下載瀏覽器。

### Q：可以在腳本中使用 await page.screenshot() 嗎？

可以，截圖是除錯的好方法。建議儲存到 `ctx.outputDir` 目錄：

```typescript
const screenshotPath = path.join(ctx.outputDir, 'step-01.png');
await page.screenshot({ path: screenshotPath });
log.info(`已儲存截圖：${screenshotPath}`);
```

### Q：腳本跑到一半失敗，如何除錯？

1. 查看終端機的錯誤訊息
2. 將錯誤訊息連同相關的 ARIA 快照一起提供給 AI
3. AI 通常能根據錯誤訊息和頁面結構找出問題

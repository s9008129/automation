# 任務腳本開發規範

> **適用對象**：開發人員、維護者、AI 生成任務腳本時的參考標準
> **重要**：AI 生成任務腳本時，必須整合本專案既有框架，不可另外生成一套獨立專案或獨立執行方式。

---

## 基本原則

1. **離線優先**：所有功能必須在無網際網路的內網環境運作
2. **入口統一**：`new-task.ps1` 建立骨架，`run-task.ps1` 執行腳本
3. **共用模組優先**：重用 `src/lib/*`，不要自行重寫 env / logger / browser / security / bootstrap
4. **Edge-first**：任務執行瀏覽器預設為 `channel: 'msedge'`
5. **結構化日誌**：使用 `runTaskEntry()` 和 `logContext()` 記錄關鍵上下文
6. **安全優先**：敏感資訊從 `.env` 讀取；輸出檔名與 URL 驗證分別重用 `safeFileName()`、`validateUrl()`
7. **正體中文**：面向使用者的訊息、提示、錯誤一律使用正體中文

---

## 腳本放置位置與命名

```text
src\{系統代碼}-{任務名稱}.ts
```

範例：

- `src\qiz-批次簽核.ts`
- `src\eip-公文簽核.ts`
- `src\vst-資料匯出.ts`

**禁止**把任務腳本放在專案根目錄，或要求一般使用者直接執行 `node` / `npm` / `npx` 指令。

## 標準腳本結構

```typescript
import { getEnv } from './lib/env.js';
import { launchTaskBrowser, closeTaskBrowser } from './lib/browser.js';
import { log, logContext, printSection } from './lib/logger.js';
import { runTaskEntry, type TaskRunContext } from './lib/task.js';

const TASK_NAME = '任務名稱';

async function runTask(context: TaskRunContext): Promise<void> {
  const systemUrl = getEnv('TARGET_URL', 'https://example.internal');

  logContext('task.inputs', {
    taskArgs: context.taskArgs,
    systemUrl,
  });

  const browserContext = await launchTaskBrowser({
    channel: 'msedge',
    headless: false,
    width: 1280,
    height: 800,
  });
  const { page } = browserContext;

  try {
    printSection('開始執行');
    log('🌐', `正在開啟：${systemUrl}`);
    await page.goto(systemUrl);

    // TODO: 在此補上任務邏輯
    log('✅', `${TASK_NAME} 任務完成`);
  } finally {
    await closeTaskBrowser(browserContext);
  }
}

const exitCode = await runTaskEntry(
  {
    taskName: TASK_NAME,
    scriptUrl: import.meta.url,
  },
  runTask
);

process.exit(exitCode);
```

## 共用 Helper 模組（`src/lib/`）

### `src/lib/task.ts` — 任務 bootstrap

- `runTaskEntry()`：統一處理 logger 初始化、`.env` 載入、runId 建立、runtime/context logging、成功 / 失敗結尾
- `TaskRunContext`：提供 `projectRoot`、`scriptPath`、`logDir`、`envPath`、`taskArgs`

### `src/lib/env.ts` — 環境變數

- `loadDotEnv()`：由 `runTaskEntry()` 統一呼叫
- `requireEnv()`：必要變數，不存在就拋中文錯誤
- `getEnv()`：可選變數，有預設值

### `src/lib/logger.ts` — 日誌與上下文

- `log()`：一般進度訊息
- `logError()`：完整錯誤堆疊
- `logContext()`：結構化上下文，供 AI / 維護者除錯
- `printHeader()` / `printSection()`：讓 console 輸出更容易讀

### `src/lib/browser.ts` — 瀏覽器模式

- `launchTaskBrowser()`：預設任務模式，直接用 Edge 執行
- `closeTaskBrowser()`：關閉由任務腳本自行啟動的 Edge
- `cdpConnect()` / `cdpDisconnect()`：只有在「必須沿用已登入的 Edge 視窗」時才使用
- `waitForNavigation()`、`takeScreenshot()`、`getNestedFrame()`：任務常用輔助

### `src/lib/security.ts` — 安全相關

- `safeFileName()`：輸出檔名安全化
- `validateTaskScriptPath()`：限制可執行腳本範圍
- `validateUrl()`（如適用）：導航 / 下載前的 URL 驗證

## 瀏覽器模式決策

### 預設：直接啟動 Edge 任務視窗

大多數 AI 生成腳本應優先使用：

```typescript
const browserContext = await launchTaskBrowser({
  channel: 'msedge',
  headless: false,
});
```

這是最容易維護、最容易交付的標準做法。

### 例外：沿用已登入的 Edge Session

只有在你**明確知道**任務必須沿用使用者當下已登入的 Edge 視窗，才改用：

```typescript
const browser = await cdpConnect();
// ...
await cdpDisconnect(browser);
```

同時，使用說明裡要清楚要求使用者先執行：

```powershell
.\launch-edge.ps1
.\run-task.ps1 src\你的腳本.ts
```

**CDP 附加模式下，不可把 `browser.close()` 當成預設收尾方式。**

## 關鍵規範（AI 生成必看）

| 項目 | ✅ 正確 | ❌ 錯誤 |
|------|---------|---------|
| 腳本位置 | `src\系統代碼-任務名稱.ts` | 放在專案根目錄 |
| 建立方式 | `new-task.ps1` 建骨架後再補完 | 叫使用者自己新建空白檔 |
| 執行入口 | `runTaskEntry()` + `run-task.ps1` | 直接教使用者跑 `node` / `npm` / `npx` |
| 共用模組 | 重用 `src/lib/*` | 自己再寫一套 `.env` / logger / bootstrap |
| 預設瀏覽器 | `channel: 'msedge'` | `channel: 'chrome'`、省略或 headless |
| 已登入 Session | 明確需求才用 `cdpConnect()` | 任何情況都強迫附加 CDP |
| 敏感資訊 | `requireEnv()` / `getEnv()` | 直接硬編碼帳號密碼 |
| 日誌 | `log()` + `logContext()` | 只有零散 `console.log()` |
| 檔名安全 | `safeFileName()` | 直接拿使用者輸入當檔名 |
| URL 驗證 | 必要時 `validateUrl()` | 對任何來源的 URL 直接導航 |

## 日誌與上下文最低要求

每支任務腳本至少應記錄：

- 本次 `taskArgs`
- 主要輸入參數（例如查詢條件、目標 URL、頁數上限）
- 關鍵步驟開始 / 完成
- 失敗當下的頁面 URL、等待中的元素或狀態
- 匯出路徑、產出筆數、總處理頁數等結果資訊

> 原則很簡單：**把 log 檔交給 AI 時，AI 應該能快速知道卡在哪一步。**

## 多頁處理模式

系統有分頁時，**必須**使用迴圈處理所有頁面：

```typescript
let currentPage = 1;
let hasNextPage = true;

while (hasNextPage) {
  log('📄', `處理第 ${currentPage} 頁...`);

  // 處理本頁邏輯
  // ...

  const nextBtn = page.getByRole('button', { name: '下一頁' });
  const isDisabled = await nextBtn.isDisabled().catch(() => true);
  const isVisible = await nextBtn.isVisible().catch(() => false);

  if (!isVisible || isDisabled) {
    hasNextPage = false;
  } else {
    await nextBtn.click();
    await page.waitForLoadState('networkidle');
    currentPage++;
  }
}

log('✅', `共處理 ${currentPage} 頁`);
```

## 建議工作流

1. 在內網先用 `collect.ps1` / `launch-edge.ps1` 蒐集同一次任務素材
2. 執行 `new-task.ps1` 建立 `src\` 骨架
3. 使用 [`docs/使用指南-Edge.md`](使用指南-Edge.md) 第 6 節的 Prompt，連同 `src\骨架.ts` 一起交給 AI
4. 把 AI 回傳結果存回 `src\`
5. 用 `run-task.ps1` 執行與驗證

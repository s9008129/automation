# Prompt A：請 AI 生成新腳本

## 🟢 使用時機

當你需要針對某個網站或系統撰寫全新的自動化腳本時，使用此 Prompt 請 AI 協助生成。

## 📋 Prompt 內容

請將以下內容複製後貼給 AI（ChatGPT、GitHub Copilot 等），並在 `【】` 內補上你的需求描述：

---

```
你是一位熟悉 Playwright + TypeScript 的 RPA 開發者。
請幫我在以下專案框架下，生成一支新的任務腳本。

## 專案規範（必須遵守）

- 腳本放在 `src/` 目錄，命名格式：`{系統代碼}-{任務名稱}.ts`
- 使用 `channel: 'msedge'`，`headless: false`
- 匯入路徑：`./lib/env.js`、`./lib/logger.js`、`./lib/browser.js`、`./lib/task.js`
- 入口固定使用 `runTaskEntry` + `run-task.ps1`
- 敏感資訊（帳號、密碼、URL）必須從 `.env` 讀取，禁止硬編碼
- 面向使用者的訊息一律用正體中文
- 不可生成獨立專案，必須整合本專案既有框架

## 任務需求

【請在這裡描述你要自動化的網站與操作步驟，例如：
  - 目標網站：https://example.com
  - 操作流程：登入 → 進入某頁面 → 下載 CSV 報表
  - 輸出：儲存到 materials/ 資料夾
】

## 參考範本

請參考 `templates/task-template.ts` 的結構生成腳本骨架，再補上實際業務邏輯。
```

---

## 📌 注意事項

- 生成後請用 `.\run-task.ps1 src\你的腳本.ts` 執行測試
- 若腳本執行失敗，請改用 [debug.md](debug.md) 的 Prompt B 協助 AI 修正

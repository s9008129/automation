# 執行任務指南

> **適用對象**：已取得任務腳本、想要直接執行自動化任務的使用者

---

## 前置確認

在執行任務腳本前，請確認：

- [ ] 已執行 `.\install.ps1` 確認安裝包完整
- [ ] 任務腳本已放在 `src\` 目錄下（例如 `src\qiz-批次簽核.ts`）
- [ ] 若需要帳號密碼，`.env` 檔案已設定（參考 `.env.example`）

---

## 基本執行方式

```powershell
.\run-task.ps1 src\腳本名稱.ts
```

### 常見範例

```powershell
# 執行 QIZ 批次簽核
.\run-task.ps1 src\qiz-批次簽核.ts

# 執行 EIP 公文簽核
.\run-task.ps1 src\eip-公文簽核.ts

# 執行並帶入自訂參數
.\run-task.ps1 src\vst-資料匯出.ts --output D:\exports

# 試跑模式（若腳本支援此參數）
.\run-task.ps1 src\qiz-批次簽核.ts --dry-run
```

---

## 執行流程說明

執行 `run-task.ps1` 時，系統會自動完成以下步驟：

1. **驗證安裝包**：確認 runtime 與相依套件完整
2. **驗證腳本路徑**：確認腳本存在且在允許範圍內
3. **解析 Node.js Runtime**：優先使用專案內建的 `runtime\node\node.exe`
4. **啟動任務腳本**：透過 `tsx` 執行 TypeScript 腳本（不需要 ts-node 或 tsc）

---

## 執行時的瀏覽器操作

大部分任務腳本使用 **Microsoft Edge** 自動執行，瀏覽器視窗會自動開啟。

### 需要人工介入的情況

某些步驟可能需要人工配合：

- **CAS 登入**：需要選擇憑證或輸入 OTP
- **驗證碼**：需要人工輸入圖形或簡訊驗證碼
- **人工確認**：腳本可能暫停並等待確認後繼續

遇到這類情況時，腳本會在 console 顯示提示，請依提示操作後等待腳本繼續。

---

## 查看執行結果

執行完成後，結果通常在以下位置：

| 類型 | 位置 |
|------|------|
| 執行日誌 | `logs\任務名稱-時間戳.log` |
| 匯出資料 | `materials\exports\` 或腳本指定目錄 |
| 截圖記錄 | 依腳本設計而定 |

## 常見錯誤處理

### 「找不到任務腳本」

```
❌ 找不到任務腳本：src\qiz-批次簽核.ts
```

**處理方式**：確認腳本檔案已放在 `src\` 目錄下，路徑拼寫正確。

---

### 「安裝包不完整」

```
❌ 找不到 scripts\resolve-node-runtime.ps1，這份安裝包不完整。
```

**處理方式**：先執行 `.\install.ps1`，若問題持續，請聯絡技術準備者重新取得完整離線包。

---

### 「無法啟動任務執行環境」

```
❌ 無法啟動任務執行環境。
❌ 找不到可用的 Node.js 執行環境。
```

**處理方式**：
1. 確認 `runtime\node\` 目錄存在且完整
2. 執行 `.\install.ps1` 確認安裝狀態
3. 如有疑問，請把 `logs\` 目錄中的日誌交給技術準備者診斷

---

### 腳本執行中斷或失敗

```
❌ 任務「QIZ-批次簽核」執行失敗
```

**處理方式**：
1. 查看 console 中的錯誤訊息
2. 查看 `logs\` 目錄中的詳細日誌
3. 把錯誤訊息和日誌提供給 AI 或技術人員協助除錯

---

### Microsoft Edge 無法啟動

**處理方式**：
1. 確認電腦已安裝 Microsoft Edge
2. 如果腳本使用 `channel: 'msedge'`，需要系統安裝的 Edge
3. Edge 需要是正常可啟動的狀態（試試手動開啟 Edge）

---

## 日誌診斷說明

每次執行任務，系統會在 `logs\` 目錄建立一份日誌檔，格式為：

```
logs\任務名稱-YYYYMMDDhhmmss.log
```

日誌包含：
- 執行時間與環境資訊
- 每個步驟的執行狀態
- 完整的錯誤堆疊（若有）

**遇到問題時，請把日誌檔交給 AI 或技術人員**，通常可以快速定位問題。

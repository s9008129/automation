# 執行任務指南

> 本文件說明如何使用 `run-task.ps1` 執行 AI 生成的自動化任務腳本。
> 若你還沒有任務腳本，請先閱讀 [AI 協作工作流](AI協作工作流.md)。

---

## 前置條件

在執行任務腳本前，請確認：

1. **已完成安裝檢查**：執行過 `.\install.ps1`，確認安裝包完整
2. **任務腳本已放置**：腳本位於 `src\` 目錄下，例如 `src\qiz-批次簽核.ts`
3. **環境變數已設定**：`.env` 檔案存在且已填入所需帳號/密碼等設定

---

## 基本用法

```powershell
.\run-task.ps1 src\{腳本名稱}.ts
```

### 範例

```powershell
# 執行批次簽核腳本
.\run-task.ps1 src\qiz-批次簽核.ts

# 執行公文簽核腳本（帶參數）
.\run-task.ps1 src\eip-公文簽核.ts --dry-run

# 查看現有腳本（不帶參數）
.\run-task.ps1
```

---

## 設定環境變數

大多數任務腳本需要帳號密碼等設定。請參照 `.env.example` 建立 `.env` 檔案：

```powershell
# 複製範本
Copy-Item .env.example .env

# 用文字編輯器開啟並填入設定
notepad .env
```

`.env` 範例內容：

```
USERNAME=your_account
PASSWORD=your_password
```

> ⚠️ `.env` 檔案包含敏感資訊，不會被上傳到版本庫，也不包含在離線包中。
> 請妥善保管，每台電腦需各自設定。

---

## 執行流程說明

`run-task.ps1` 執行時會進行以下步驟：

1. **驗證腳本路徑**：確認是 `.ts` 檔案且存在
2. **安全性檢查**：確認腳本位於專案目錄內
3. **載入 Node.js Runtime**：優先使用 `runtime\node\`，再退回系統安裝
4. **設定瀏覽器路徑**：若有離線瀏覽器包，自動設定
5. **執行腳本**：透過 `tsx` 執行 TypeScript 腳本
6. **傳遞 exit code**：腳本成功則 exit 0，失敗則傳遞對應 exit code

## 常見錯誤與排除

### ❌ 找不到任務腳本

```
❌ 找不到任務腳本：src\qiz-批次簽核.ts
```

**原因**：腳本路徑錯誤或腳本尚未建立  
**解決**：
- 執行 `.\run-task.ps1`（不帶參數）查看現有腳本列表
- 確認腳本已存放在 `src\` 目錄下
- 若尚無腳本，請先依 [AI 協作工作流](AI協作工作流.md) 生成腳本

---

### ❌ 腳本必須是 .ts 檔案

```
❌ 腳本必須是 .ts 檔案，不支援「.js」。
```

**原因**：傳入的不是 TypeScript 腳本  
**解決**：確認傳入的是 `.ts` 檔案

---

### ❌ 基於安全考量，只允許執行專案資料夾內的腳本

```
❌ 基於安全考量，只允許執行專案資料夾內的腳本。
```

**原因**：腳本路徑超出專案目錄範圍  
**解決**：將腳本放在 `src\` 目錄下，使用相對路徑

---

### ❌ 無法啟動任務執行環境

```
❌ 無法啟動任務執行環境。
```

**原因**：Node.js Runtime 缺失或版本不符  
**解決**：
- 執行 `.\install.ps1` 確認安裝包完整
- 若 `runtime\node\` 目錄不存在，代表工具包不完整
- 請聯絡技術準備者重新提供完整工具包

---

### ❌ 腳本執行中途失敗（任務邏輯錯誤）

終端機會顯示如下錯誤：

```
❌ 【批次簽核】執行失敗：找不到元素：按鈕「確認簽核」

--- 錯誤堆疊（供 AI 分析用）---
...
---

請將以上錯誤訊息提供給 AI 協助排查。
```

**解決**：
1. 複製整段錯誤訊息
2. 同時提供相關的 ARIA 快照（`materials\...\ aria-snapshots\*.txt`）
3. 提供給 AI 協助分析與修正腳本

---

### ❌ 缺少必要的環境變數

```
❌ 缺少必要的環境變數：PASSWORD
請複製 .env.example 為 .env 並填入正確設定。
```

**解決**：建立並填寫 `.env` 檔案（參考上方「設定環境變數」章節）

---

## 日誌與輸出

任務腳本執行時，輸出資訊會顯示在終端機上。
部分腳本也會在 `logs\` 目錄下產生執行紀錄：

```
logs\
└── 20260318111335_批次簽核\
    ├── execution.log    ← 執行紀錄
    └── *.png            ← 執行過程截圖（依腳本而定）
```

---

## 其他說明

- 任務執行模式下，腳本會自動啟動 Microsoft Edge 視窗並在完成後關閉
- 執行中途可按 `Ctrl+C` 中止，但瀏覽器視窗可能需要手動關閉
- 若腳本需要使用系統 Edge，請確認 Microsoft Edge 已安裝在電腦上

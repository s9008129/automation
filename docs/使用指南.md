## 0. 前言：為什麼選 Playwright，不選 Selenium？

> 你不需要懂這兩個工具的技術細節。
> 下面用三個比喻說明，看完你就知道為什麼這個專案做了這個選擇。

### 🎯 比喻一：找按鈕的方式不同

**Selenium 的做法**像是有人告訴你：「往右走 300 步、往下走 200 步，那裡有一個按鈕，按下去。」

問題是：辦公室重新裝潢過（網頁改版），或你的螢幕比較小（解析度不同），那個按鈕就不在那個位置了。你照著走，結果按到空氣。

**Playwright 的做法**像是直接告訴你：「找到寫著『查詢』兩個字的按鈕，按下去。」

不管辦公室怎麼改裝潢、你的螢幕多大多小，只要那個按鈕還在頁面上，它就找得到。

### 🔄 比喻二：網頁改版時的抗震能力

內網系統常常改版——換個版型、調個選單位置、多一欄少一欄。

用座標和固定路徑去找元素的方式，就像把家具位置全部背下來，然後閉著眼走路。**家具一搬，就撞牆。**

Playwright 用的是頁面上的「文字」和「語意結構」（例如：這是一個按鈕、這是一個輸入欄位）來找目標。就像你走進一間重新裝潢的辦公室，不用背家具位置，直接看門牌找房間就好。**家具搬了，門牌還在。**

### 🔧 比喻三：少一個要維護的東西

Selenium 需要一個叫「driver」的東西，作用像是瀏覽器的鑰匙。**瀏覽器更新了，鑰匙也要換。** 版本沒對上，工具就打不開瀏覽器。

在內網環境，你不一定能隨時上網下載新的 driver。結果就是：瀏覽器自動更新了，driver 沒跟上，整個工具就不能用了。

Playwright 不需要這種 driver 對版。**鑰匙跟鎖是一體的，不用自己配。** 這對離線環境來說省掉很多麻煩。

### 📋 三句話總結

| 痛點 | Selenium | Playwright |
|------|----------|------------|
| 找按鈕的方式 | 靠座標走路，位置一變就按到空氣 | 靠文字和語意找，位置變了照樣找到 |
| 網頁改版 | 路徑全壞，要重新調整 | 只要元素文字還在，通常不用改 |
| 瀏覽器更新 | driver 要配對版本，沒對上就不能用 | 不需要額外 driver，離線包自帶 |

> **所以 RPA-Cowork 選擇 Playwright：更穩定、更省事、更適合離線環境。**

## 1. 蒐集素材（3 步驟）

> 🎯 **目標：把你在 Edge 上看到的頁面，保存成一包素材，等一下交給 AI。**

### 步驟 1-1：檢查工具包

打開 PowerShell，進入工具資料夾，執行：

```powershell
.\install.ps1
```

**你會看到：**

```
============================================================
  🔍 RPA-Cowork：檢查離線工具包
============================================================

✅ runtime\node\ ............ 存在
✅ node_modules\ ............ 存在
✅ .playwright-browsers\ .... 存在
✅ .env.example ............. 存在

🎉 工具包完整，可以開始使用。
```

> ✅ 全部通過 → 繼續下一步。
>
> ❌ 出現紅字 → 把 `logs\` 資料夾裡最新的 `.log` 檔交給幫你準備工具包的人，請對方重新打包。不要自己安裝 Node.js 或 npm。

### 步驟 1-2：開啟 Edge 專用視窗

```powershell
.\launch-edge.ps1
```

**這時會跳出一個新的 Edge 視窗。** 在這個視窗裡做兩件事：

1. 登入你要操作的目標網站
2. 打開你要蒐集的那個頁面

> ⚠️ **不要關掉這個 Edge 視窗。** 下一步需要透過它來蒐集素材。

### 步驟 1-3：開始蒐集

```powershell
.\collect.ps1 --browser edge
```

**你會看到選單：**

```
請選擇模式：
  [1] 互動模式（推薦新手）
  [2] 全自動模式
  [3] 快照模式
  [4] 錄製模式
```

**輸入 `1` 然後按 Enter。**

系統接著會問你兩個問題，照這樣填寫：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `📌 請輸入專案名稱：` | `tax-cloud` | 用英文代號，代表哪個系統 |
| `📌 請輸入本次任務名稱：` | `批次簽核` | 用中文，描述這次要做什麼 |

系統會自動連上你的 Edge 頁面，然後進入互動模式。

### 🔄 互動模式完整流程

互動模式分成 **兩個階段**，你可以隨時在兩個階段之間來回切換：

```
階段 1/2：頁面蒐集          階段 2/2：流程錄製
┌─────────────────┐        ┌─────────────────┐
│ 擷取 ARIA 快照   │  ←→    │ 錄下你的操作流程  │
│ 擷取截圖         │        │ （會開新視窗）    │
└─────────────────┘        └─────────────────┘
        ↓                          ↓
   蒐集完所有頁面後          錄製完所有流程後
        ↓                          ↓
        └──────── 結束並產生報告 ────┘
```

#### 階段 1：頁面蒐集

進入互動模式後，你會先看到：

```
  ────────────────────────────────────────────────────────────
  階段 1/2：頁面蒐集（ARIA-first）
  請先到已登入的 Edge，把畫面切到你要蒐集的頁面，再回來依選單操作。
  ────────────────────────────────────────────────────────────
```

接著出現選單：

```
  頁面蒐集階段：接下來要做什麼？
    [1] 蒐集目前 Edge 頁面（第 1 頁）
    [2] 切換到錄製階段
    [3] 結束並產生報告
```

**輸入 `1`** → 系統會偵測你 Edge 目前的頁面，顯示：

```
  目前準備蒐集第 1 頁
    - Edge 頁面標題：財稅雲端平台入口網
    - Edge 頁面 URL：https://eip.tax.example.gov.tw/login

  請先確認 Edge 現在顯示的就是你要蒐集的頁面，再輸入下面資訊。
```

然後問你兩個問題：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `頁面名稱（例如：01-login-page）：` | `01-login-page` | 英文編號，方便辨識 |
| `頁面說明（Enter 使用預設值）：` | 直接按 Enter | 系統會用頁面標題當說明 |

蒐集完成後你會看到：

```
  ✅ 這一頁的素材已完成
    - 頁面名稱：01-login-page
    - 頁面標題：財稅雲端平台入口網
    - ARIA 快照：aria-snapshots/01-login-page.txt
    - 截圖：screenshots/01-login-page.png
    - 目前累計：1 頁面 / 0 錄製 / 0 錯誤
```

接著出現下一步選單：

```
  本頁完成，下一步要做什麼？
    [1] 繼續蒐集下一個頁面
    [2] 進入錄製階段
    [3] 結束並產生報告
```

> 💡 **要蒐集多個頁面：** 在 Edge 裡切換到下一個頁面 → 回到 PowerShell 選 `1` → 重複以上步驟。
> 每蒐集一頁，系統會自動把 ARIA 快照和截圖存好。

#### 階段 2：流程錄製

選擇「切換到錄製階段」後，你會看到：

```
  ────────────────────────────────────────────────────────────
  階段 2/2：流程錄製
  這個階段會開啟一個新的 Playwright 錄製視窗，和你原本的 Edge 分開。
  請在新視窗完成操作，錄製結束時直接關閉該視窗即可。
  ────────────────────────────────────────────────────────────
```

選單：

```
  錄製階段：接下來要做什麼？
    [1] 開始新的錄製
    [2] 返回頁面蒐集階段
    [3] 結束並產生報告
```

**輸入 `1`** → 系統會問你錄製資訊：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `錄製名稱（Enter 使用預設值）：` | 直接按 Enter | 系統會自動命名 |
| `起始 URL（Enter 使用預設值）：` | 直接按 Enter | 系統會用你剛才蒐集的頁面 URL |
| `操作說明：` | `登入→進入批次簽核→全選→F6處理` | 描述你要錄下什麼操作 |

接著一個 **新的瀏覽器視窗** 會跳出來。在這個新視窗裡操作你要錄製的流程：

1. 在新視窗裡完成你的操作（例如：登入 → 進入批次簽核 → 全選 → F6 批次處理）
2. 操作完畢後，**直接關閉新視窗**
3. 回到 PowerShell，系統會自動保存錄製檔

```
  ✅ 錄製檔已納入本次成果
    - 流程名稱：01-batch-flow
    - 錄製檔：recordings/01-batch-flow.js
    - 目前累計：1 頁面 / 1 錄製 / 0 錯誤
```

> ⚠️ **錄製視窗和你原本的 Edge 是分開的。** 錄製視窗關掉不會影響你已登入的 Edge。

#### 結束蒐集

不管在哪個階段，選 **「結束並產生報告」** 就會結束整個流程，產生摘要報告。

### ✅ 蒐集完成

你會在資料夾裡看到這樣的結構：

```text
materials\
└─ 20260318143000_批次簽核\
   ├─ aria-snapshots\       ← 頁面結構（AI 的核心素材）
   │   ├─ 01-login-page.txt
   │   ├─ 02-homepage.txt
   │   ├─ 03-qiz201q.txt
   │   └─ 04-batch-approval.txt
   ├─ screenshots\          ← 畫面截圖
   │   ├─ 01-login-page.png
   │   ├─ 02-homepage.png
   │   ├─ 03-qiz201q.png
   │   └─ 04-batch-approval.png
   ├─ recordings\           ← 操作錄影
   │   └─ 01-batch-flow.js
   ├─ metadata.json         ← 背景資訊
   └─ summary-report.md     ← 摘要報告
```

> **這包素材就是等一下要上傳給 AI 的附件。保持整包不動，不要拆散。**

### 💡 為什麼這些素材這麼重要？

> 以下是一個真實案例的回饋，說明 AI 如何利用你蒐集的每種素材。

| 你蒐集的素材 | AI 從裡面看到什麼 | 為什麼重要 |
|------------|------------------|-----------|
| ARIA 快照（`.txt`） | 頁面結構：哪些按鈕叫什麼名字、表單有幾個欄位、iframe 層級關係 | AI 靠這個寫出正確的「找按鈕」指令，不會按到空氣 |
| 截圖（`.png`） | 畫面實際長相：按鈕是否可以按、有沒有分頁、目前顯示幾筆資料 | AI 靠這個確認邏輯：例如「看到共 66 筆 / 2 頁，所以要寫迴圈處理所有頁」 |
| 錄製檔（`.js`） | 你的操作順序：先點哪裡、再填什麼、最後按哪個按鈕 | AI 靠這個理解流程，但會**主動找出錯誤**並修正（例如找出硬編碼的網址、漏掉的分頁處理） |
| `metadata.json` | 蒐集環境：用的什麼瀏覽器、什麼系統、什麼時間 | AI 靠這個決定腳本要用 Edge 還是 Chrome、時區設定等 |

> 📌 **重點：素材蒐集得越完整，AI 產出的腳本品質越高。** 不要只蒐集一個頁面就急著給 AI——把流程中會經過的每個重要頁面都蒐集下來。

## 2. 請 AI 幫你寫腳本（3 步驟）

> 🎯 **目標：讓 AI 把你蒐集的素材，變成一支可以直接執行的自動化腳本。**

### 步驟 2-1：建立腳本骨架

在 PowerShell 執行：

```powershell
.\new-task.ps1
```

系統會問你兩個問題：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `📌 請輸入系統代碼：` | `tax-cloud` | 英文，識別哪個系統 |
| `📌 請輸入任務名稱：` | `批次簽核` | 中文，描述要做什麼 |

完成後你會看到：

```
✅ 腳本已建立：src\tax-cloud-批次簽核.ts
```

> 你不用看懂骨架檔的內容。它的作用是告訴 AI：**「在這個框架裡幫我補完，不要另建新專案。」**

### 步驟 2-2：上傳附件給 AI

打開 [Claude](https://claude.ai)，確認使用 **Claude Sonnet 4.6** 模型，開始新對話。

**點迴紋針圖示 📎，依序上傳以下檔案：**

| # | 上傳什麼 | 到哪裡找 |
|---|---------|---------|
| 1 | 骨架檔 `src\tax-cloud-批次簽核.ts` | 工具資料夾 → `src\` |
| 2 | 所有 `.txt` 檔 | `materials\20260318143000_批次簽核\aria-snapshots\` |
| 3 | 所有 `.png` 檔 | `materials\20260318143000_批次簽核\screenshots\` |
| 4 | `metadata.json` | `materials\20260318143000_批次簽核\` |
| 5 | 所有 `.js` 檔（錄製模式的產出） | `materials\20260318143000_批次簽核\recordings\` |

> ⚠️ **絕對不要上傳 `.env` 檔案。** `.env` 裡面有你的帳號密碼。

### 步驟 2-3：貼上 Prompt 給 AI

上傳完附件後，**複製下方 Prompt 範本，貼到 Claude 的訊息框送出。**

```
📌 使用方式：
   看到 ⬅ 的那一行 → 一定要改成你自己的內容
   沒有 ⬅ 的那一行 → 原封不動保留，直接送出
```

> 範本裡已經預填了一個「財稅平台批次簽核」的範例。
> 你只要把有 `⬅` 的行改成你自己的狀況，其他全部保留。

---

### 🟢 Prompt A：請 AI 生成新腳本

````text
我已上傳這次任務的全部附件，包含：
- 骨架檔：src\tax-cloud-批次簽核.ts          ⬅ 改成你的骨架檔名
- ARIA 快照（頁面結構）
- 截圖（畫面狀態）
- 錄製檔（操作流程）
- metadata.json（任務背景資訊）

請以我上傳的骨架檔為基礎，幫我補完這支腳本。

我要完成的事情：                          ⬅ 以下全部改成你要自動化的操作步驟
1. 登入財稅雲端平台入口網
2. 等 SSO 登入完成後自動跳轉到 EIP 首頁
3. 點選電子表單，開啟批次簽核功能（會跳出新視窗）
4. 在批次簽核頁面選擇功能代碼
5. 點「我的批次簽核」載入待辦清單
6. 點「全選」→ 按「F6批次處理」
7. 如果有多頁，重複步驟 6 直到所有頁面處理完畢
8. 確認待辦清單顯示「共 0 筆」

特別注意：                                ⬅ 如果有要強調的事項可以加這段
````

### 🔍 真實案例：AI 收到素材後做了什麼

> **以下是一個真實的互動案例，讓你了解 AI 分析素材的過程。**

**情境：** 某位同仁用互動模式蒐集了「財稅雲端平台批次簽核」的 4 個頁面和 1 段錄製，上傳給 AI。

## 4. 遇到問題怎麼辦

> **按照你看到的症狀，直接找對應的解法。**

---

### ❌ `install.ps1` 顯示紅色錯誤

**原因：** 離線包不完整。

**解法：** 把 `logs\` 最新的 `.log` 交給幫你準備工具包的人，請對方重新打包。不要自己安裝 Node.js 或 npm。

---

### ❌ `launch-edge.ps1` 沒有反應

**排查：**

1. 確認這台電腦有安裝 Microsoft Edge
2. 確認你在完整的工具資料夾裡執行（不是只有 `.ps1` 檔案）
3. 重新執行一次 `.\launch-edge.ps1`

還是不行 → 把 `logs\` 最新日誌交給準備工具包的人。

---

### ❌ `collect.ps1 --browser edge` 顯示「找不到頁面」

**排查：**

1. 確認你是用 `.\launch-edge.ps1` 開出來的那個 Edge 視窗（不是自己另外開的）
2. 確認那個 Edge 視窗還開著
3. 確認你已經在 Edge 裡登入了目標網站
4. 確認指令有加 `--browser edge`

---

### ❌ AI 叫你 `npm install` 或 `npx`

**原因：** AI 不知道專案已有框架，以為你要從零開始。

**解法：** 用 [Prompt C](#-prompt-cai-回覆不對叫你-npm-install-或另建專案) 補上框架檔案再問一次。

---

### ❌ `run-task.ps1` 顯示「缺少 .env」

**解法：**

```powershell
Copy-Item .env.example .env
notepad .env
```

填入 Claude 說需要的欄位值，存檔後重新執行。

---

### ❌ 腳本跑到一半停住或紅色錯誤

**解法：** 按照 [執行失敗的處理流程](#執行失敗的處理流程) 蒐集錯誤資訊，用 [Prompt B](#-prompt-b請-ai-修正失敗的腳本) 請 AI 修正。

---

### ❌ PowerShell 顯示「已停用指令碼執行」

**解法：** 用系統管理員身分打開 PowerShell，執行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

單位有更嚴格的政策 → 請依內部規範處理。

# RPA-Cowork 使用指南

> 這是一份給一般使用者的**唯一正式使用指南**。
> 你不用再分辨 Edge 版、AI 協作版、FAQ 版、執行版。
>
> 只要記住這條主線：
>
> **`install.ps1 -> launch-edge.ps1 -> collect.ps1 --browser edge -> new-task.ps1 -> 上傳附件給 AI -> run-task.ps1`**

---

## 1. 先記住這 4 件事

1. **本指南主線以 Microsoft Edge 為主**
   如果你們單位仍指定 Chrome，請把 `launch-edge.ps1` 換成 `launch-chrome.ps1`，把 `collect.ps1 --browser edge` 換成 `collect.ps1`。

2. **一般使用者不用先研究 npm / npx / Node.js**
   如果畫面跳出找不到 `node`、`npm`、`npx`，優先視為**離線包不完整**，不是要你自己補安裝。

3. **AI 看不到你的整個 repo**
   AI 只看得到你上傳的附件。所以真正的上下文，不是靠貼一大串規則，而是靠你上傳的**骨架檔 + 同一次任務附件**。

4. **不要上傳 `.env`**
   `.env` 可能有帳號、密碼或 token。需要給 AI 的，是骨架、素材、日誌，不是敏感資料。

---

## 2. 先照這 3 步把素材抓回來

### 步驟 1：先跑 `install.ps1`

```powershell
.\install.ps1
```

這一步是在**檢查離線包是否完整**，不是上網安裝器。

如果失敗，最快的做法就是：

1. 到 `logs\` 找最新的 `.log`
2. 把 log 交給準備工具包的人
3. 請對方重新確認離線包是否完整

### 步驟 2：開啟 Edge 專用視窗

```powershell
.\launch-edge.ps1
```

接著請做 3 件事：

1. 在 Edge 裡登入目標網站
2. 打開你要蒐集的頁面
3. 先不要把這個 Edge 視窗關掉

### 步驟 3：開始蒐集

```powershell
.\collect.ps1 --browser edge
```

第一次使用，建議先選**互動模式**。

你可以先這樣記：

- `1` = **互動模式**：最適合新手，系統一步一步帶你蒐集
- `3` = **快照模式**：只先確認工具有沒有真的接上目前頁面
- `4` = **錄製模式**：你已經知道流程，想直接錄操作

### 最短範例

假設你現在在「案件查詢」頁：

1. 跑 `install.ps1`
2. 跑 `launch-edge.ps1`
3. 在 Edge 裡登入，並打開案件查詢頁
4. 跑 `collect.ps1 --browser edge`
5. 在選單輸入 `1`（互動模式）
6. 依提示輸入：
   - `專案名稱`：`case-portal`
   - `本次任務名稱`：`案件查詢`
7. 先蒐集查詢頁，再蒐集結果頁，需要時再錄製一段操作

你可以把這整段想成：

**「先把頁面證據裝進同一個材料袋，等一下再交給 AI。」**

---

## 3. 蒐集完成後，你會拿到什麼

每次蒐集完成後，通常會在 `materials\` 下面看到像這樣的資料夾：

```text
materials\
└─ YYYYMMDDhhmmss_案件查詢\
   ├─ aria-snapshots\
   ├─ screenshots\
   ├─ recordings\
   ├─ metadata.json
   └─ summary-report.md
```

你可以這樣理解：

- `aria-snapshots\`：給 AI 看「頁面結構」
- `screenshots\`：給 AI 看「畫面長相」
- `recordings\`：給 AI 看「你怎麼操作」
- `metadata.json`：補充時間、頁面、模式等資訊

**重點只有一個：拿給 AI 的附件，要盡量來自同一次任務。**
不要把別次任務的截圖、錄製、日誌混在一起。

如果你只是要補抓目前失敗頁面的狀態，也可以快速重跑：

```powershell
.\collect.ps1 --snapshot --browser edge
```

## 4. 請 AI 幫你做腳本：照這 3 步

### 步驟 1：先用 `new-task.ps1` 開骨架

```powershell
.\new-task.ps1
```

它會問你兩個東西：

- `系統代碼`：例如 `case`、`eip`、`qiz`
- `任務名稱`：例如 `案件查詢`、`公文匯出`

完成後，你會拿到一個像這樣的檔案：

```text
src\case-案件查詢.ts
```

你只要這樣理解就好：

- `templates\task-template.ts` = 工具內建範本
- `new-task.ps1` = 幫你把範本變成這次任務專用骨架的入口
- `src\你的腳本.ts` = 你真正要交給 AI、最後也要拿來執行的檔案

所以你**不用先看懂 `task-template.ts` 每一行**。
你要做的，是把 `new-task.ps1` 產生出來的 `src\你的腳本.ts` 交給 AI 補完。

### 步驟 2：先上傳附件，不要先貼長文

#### 如果你要 AI 生成新腳本，先上傳這些

- `src\你的腳本.ts`
- 同一次任務的 `aria-snapshots\`
- 同一次任務的 `screenshots\`
- 同一次任務的 `recordings\`
- `metadata.json`

#### 如果你要 AI 除錯現有腳本，再多上傳這些

- 目前失敗的 `src\你的腳本.ts`
- `logs\` 相關日誌
- PowerShell 完整錯誤訊息
- 必要時再補一次最新的 `--snapshot` 結果

#### 如果 AI 常常寫歪，再多補兩份就好

- `templates\task-template.ts`
- `run-task.ps1`

這兩份的用途很單純：
讓 AI 知道你要的是**補完既有框架**，不是另外建立一個新 Playwright 專案。

### 步驟 3：貼短 Prompt，不要貼一大坨規則

下面這兩段就是現在建議的一般使用者版本。
核心原則是：

- **先上傳附件**
- **再用短 Prompt 鎖定目標**
- **讓 AI 依附件回覆，不要自己猜**

### 情境 A：我要生成新腳本

> 你真正要改的，通常只有：腳本名稱、要完成的事情。

````text
我已上傳同一次任務的附件。請不要假設你看得到整個 repo。

請以我上傳的 `src\case-案件查詢.ts` 骨架為準，補完這支腳本，讓我最後可以用 `.\run-task.ps1 src\case-案件查詢.ts` 執行。

固定規則：
- 不要另外建立新專案
- 保留骨架裡既有 import / runTaskEntry / Edge 設定
- 不要要求 npm install、npx、npm run
- 帳密不要硬編碼；需要時改成 `.env`
- 如果附件不夠，先列缺什麼，不要猜

我要完成：
登入後進入案件查詢頁，輸入案件編號 A123456，按「查詢」，再打開第一筆案件明細。

請依序回我：
1. 你從附件看懂了什麼
2. 還缺什麼
3. 可直接覆蓋的 `src\case-案件查詢.ts`
4. 需要新增的 `.env` 欄位（沒有就寫無）
5. 執行步驟
6. 成功檢查方式
````

### 情境 B：我要除錯現有腳本

> 你真正要改的，通常只有：腳本名稱、現在卡在哪裡、實際錯誤。

````text
我已上傳同一次任務的附件。請不要假設你看得到整個 repo。

請以我上傳的 `src\case-案件查詢.ts` 為準，找出問題並直接給我可覆蓋的修正版，讓我最後可以用 `.\run-task.ps1 src\case-案件查詢.ts` 重新執行。

固定規則：
- 不要另外建立新專案
- 保留既有骨架、import、runTaskEntry、Edge 設定
- 不要要求 npm install、npx、npm run
- 帳密不要硬編碼；需要時改成 `.env`
- 如果附件不夠，先列缺什麼，不要猜

我要修正的問題：
在案件查詢頁輸入 A123456 後，按下「查詢」就失敗，沒有成功打開第一筆明細。

實際錯誤：
PowerShell 顯示 `Timeout 30000ms exceeded`。

請依序回我：
1. 最可能原因（1~3 個，請附證據）
2. 還缺什麼
3. 可直接覆蓋的 `src\case-案件查詢.ts`
4. 需要新增或調整的 `.env` 欄位（沒有就寫無）
5. 重新執行步驟
6. 修正後成功訊號
7. 如果還失敗，下次要補哪些附件
````

### 如果 AI 還是寫歪，直接回這句就好

如果 AI 回成「請先 `npm install`」或直接另開一個新專案，你不用重新解釋一大堆，只要：

1. 再補上 `templates\task-template.ts` 和 `run-task.ps1`
2. 回這句：

````text
請不要另外建立新專案，請以我上傳的骨架檔與 run-task.ps1 為準，直接回覆可覆蓋的 `src\腳本名稱.ts`。
````

## 5. 拿到 AI 結果後怎麼執行

1. 把 AI 回傳的內容，覆蓋到 `src\你的腳本.ts`
2. 如果 AI 有列出新的 `.env` 欄位，先做：

   ```powershell
   Copy-Item .env.example .env
   notepad .env
   ```

3. 如果 AI 明確要求要沿用已登入的 Edge 視窗，先跑：

   ```powershell
   .\launch-edge.ps1
   ```

4. 執行腳本：

   ```powershell
   .\run-task.ps1 src\你的腳本.ts
   ```

### 最短範例

```powershell
Copy-Item .env.example .env
notepad .env
.\run-task.ps1 src\case-案件查詢.ts
```

如果執行失敗，不要急著改一堆程式碼。先把這些東西留好：

- PowerShell 完整錯誤訊息
- `logs\` 最新日誌
- 失敗頁面的最新快照（必要時再跑一次 `--snapshot`）

然後回到上面「情境 B：我要除錯現有腳本」那段，再交給 AI。

## 6. 最常見的 7 個問題

### 1. `install.ps1` 失敗

通常代表**離線包不完整**。
請把 `logs\` 最新 `.log` 交給準備工具包的人，不要自己補裝 Node.js 或 npm。

### 2. `launch-edge.ps1` 跑不起來

先確認：

- 這台電腦有安裝 Microsoft Edge
- 你是在完整工具資料夾裡執行
- 重新跑一次是否恢復正常

如果還是不行，一樣把 `logs\` 最新啟動日誌交出去。

### 3. `collect.ps1 --browser edge` 抓不到頁面

先確認：

- 你是不是用 `launch-edge.ps1` 開的那個 Edge 視窗
- 那個 Edge 視窗是不是還開著
- 你是不是已登入，而且已打開目標頁面
- 指令裡有沒有真的加上 `--browser edge`

### 4. AI 叫我 `npm install` 或 `npx`

這代表 AI 沒有抓到本專案上下文。
做法：

1. 不要照做
2. 補上 `templates\task-template.ts` 和 `run-task.ps1`
3. 回 AI：

   ````text
   請不要另外建立新專案，也不要要求 npm install / npx。請直接以我上傳的骨架檔回覆可覆蓋的 `src\腳本名稱.ts`。
   ````

### 5. `run-task.ps1` 說缺少 `.env` 欄位

先做：

```powershell
Copy-Item .env.example .env
notepad .env
```

把 AI 列出需要的欄位補進去，再重新執行。

### 6. 腳本執行到一半就停住

先保留：

- PowerShell 完整錯誤訊息
- `logs\` 最新日誌
- 失敗頁面的最新快照

然後用本指南第 4 節的除錯範例，再交給 AI 一次。

### 7. PowerShell 說「已停用指令碼執行」

請開啟 PowerShell（建議系統管理員）後執行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

如果你們單位有更嚴格的政策，請依內部規範處理。

---

## 7. 一句話記住

**先用 Edge 把同一次任務的素材抓完整，再用 `new-task.ps1` 建骨架，然後把「骨架檔 + 同一次附件」交給 AI，最後只用 `run-task.ps1` 執行。**

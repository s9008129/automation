# 內部網路網頁素材蒐集工具使用指南

> 這份手冊先只回答一件事：
> **怎麼在有網路的電腦先把工具準備好，然後把整個資料夾搬到內網直接用。**
>
> 如果你只想先把事情做完，不想先研究技術名詞，請直接看第 1 節和第 4 節。
>
> 以下主線以 **Windows 11 + PowerShell 7.x** 為主。

## 1. 最短版本：先看這裡就夠了

### 如果你是幫大家準備工具包的人（有網路）

請在**可上網的電腦**、專案原始資料夾裡執行：

```powershell
.\scripts\prepare-offline-bundle.ps1
```

這一步會幫你把需要帶進內網的東西先準備好，包含：

- 工具自己的執行引擎：`runtime\node\`
- 工具需要的零件資料夾：`node_modules\`
- 工具需要的瀏覽器元件：`.playwright-browsers\`
- 給一般使用者直接執行的入口檔：
  - `install.ps1`
  - `launch-chrome.ps1`
  - `launch-edge.ps1`
  - `collect.ps1`

完成後，請把**整個產出的資料夾**完整交給內網使用者。

---

### 如果你是內網裡直接使用的人

你通常只要照這 3 步做（**標準 Chrome 流程**）：

```powershell
.\install.ps1
.\launch-chrome.ps1
.\collect.ps1
```

如果你們單位指定使用 **Microsoft Edge**，請改用：

```powershell
.\install.ps1
.\launch-edge.ps1
.\collect.ps1 --browser edge
```

如果你們單位平常就是固定使用 **Microsoft Edge**，建議直接改看獨立版：[`docs/使用指南-Edge.md`](使用指南-Edge.md)。

順序請照著來：

1. `install.ps1`
   - 先檢查這份工具包是不是完整可用
2. `launch-chrome.ps1` / `launch-edge.ps1`
   - 打開專用瀏覽器視窗（預設 Chrome；指定 Edge 時改用 Edge 入口）
3. `collect.ps1`
   - 開始蒐集網站素材；若前一步使用 Edge，請加上 `--browser edge`

> 如果第 1 步失敗，**不要自己研究 npm、Node.js 或安裝套件**。
> 這通常代表你拿到的不是完整工具包，請直接找幫你準備工具包的人重新打包。

---

## 3. 我是有網路的準備者：照這幾步做

這一段是給**幫大家準備完整工具包的人**看的。
如果你只是內網裡的使用者，請直接跳到下一節。

### 3.1 你真正要用的是哪一支？

請直接用這支：

```powershell
.\scripts\prepare-offline-bundle.ps1
```

你可以把它理解成：

**「把原始專案整理成一份可以直接搬進內網的完整工具包」**

這支腳本是給準備者用的，不是給一般內網使用者用的。

---

### 3.2 你要怎麼做？

#### 步驟 1：在有網路的 Windows 電腦打開專案資料夾

請先進到這份工具的原始資料夾，再開 PowerShell。

如果這台電腦本來就是技術準備機，通常會有完整原始專案與可上網環境。

#### 步驟 2：執行打包指令

```powershell
.\scripts\prepare-offline-bundle.ps1
```

請等它跑完。

這一步不是只複製幾個檔案而已，它會幫你把內網需要的關鍵內容先備齊。

---

### 3.3 這一步會幫你準備哪些東西？

準備完成後，完整工具包裡通常會有這些關鍵內容：

#### 1. `runtime\node\`
這是工具自己的**內建執行引擎**。

你可以把它想成：

- 工具自己附帶的小引擎
- 讓內網使用者通常不必再自己裝對應環境

#### 2. `node_modules\`
這是工具需要的**零件資料夾**。

裡面會包含這個工具運作需要的套件，例如：

- `playwright`
- `tsx`
- `typescript`
- `@types/node`

你不需要記這些名字，但要知道：

> **完整工具包必須把這個資料夾一起帶進內網。**

#### 3. `.playwright-browsers\`
這是工具需要的**瀏覽器元件資料夾**。

它不是你平常日常使用的 Chrome 或 Edge，
而是這個工具自己要用到的 Playwright Chromium runtime。
即使你現場選擇用 Edge 蒐集，這個資料夾也一樣要跟著離線包一起交付。

#### 4. Windows 直接使用入口
內網使用者最常會用到的是：

- `install.ps1`
- `launch-chrome.ps1`
- `launch-edge.ps1`
- `collect.ps1`

你交付時，請確保這些檔案都在。

---

### 3.4 打包完成後，工具包通常會在哪裡？

如果你沒有另外指定位置，通常會產生在類似這種路徑：

```text
output\offline-bundles\web-material-collector-offline-時間戳記
```

你可以把這個新產生的資料夾理解成：

**「這就是要交給內網使用者的完整工具包」**

---

### 3.5 你要怎麼交給內網使用者？

最重要的原則只有一個：

> **請交整個資料夾，不要只交幾個 `.ps1` 檔案。**

請這樣做：

- 直接複製整個資料夾
- 或先壓縮成 zip，再到內網電腦解壓縮

請不要這樣做：

- 不要只挑 `install.ps1` 給對方
- 不要漏掉子資料夾
- 不要把資料夾拆散後再重組

---

### 3.6 交付前，最簡單的自我檢查方式

你可以進到打包出來的那份資料夾，再執行：

```powershell
.\install.ps1
```

如果這一步檢查能通過，通常就代表：

- 工具包夠完整
- 內網使用者拿到後，成功機率會高很多

---

### 3.7 你可以怎麼跟內網使用者說？

最簡單的交接話術如下：

1. 先跑 `install.ps1`
2. 預設再跑 `launch-chrome.ps1`
3. 如果你們單位指定用 Edge，就改跑 `launch-edge.ps1`，並在蒐集時加上 `--browser edge`
4. 登入網站後跑 `collect.ps1`
5. 如果第一步失敗，不要自己研究，直接回來找我

這樣就夠了。

---

## 4. 我是內網使用者：照這幾步做

這一段是給**拿到完整工具包、要在內網直接使用的人**看的。

### 4.1 先記住：你通常不用自己安裝技術環境

如果你拿到的是別人準備好的完整工具包，通常你不需要自己研究：

- npm
- Node.js
- 套件安裝
- 瀏覽器元件
- 技術環境設定

你只要照流程執行即可。

---

### 4.2 第一步：先把整個資料夾放好

請先把對方給你的**整個工具資料夾**放到你的電腦上。

建議放在：

- 桌面
- 文件夾
- D 槽工作區
- 任何你有權限操作的位置

請不要：

- 不解壓縮就直接從壓縮檔裡執行
- 只複製其中幾個檔案出來
- 把原本的資料夾拆散

最安全的做法是：

> **先把整個資料夾完整放好，再開始操作。**

---

### 4.3 第二步：先跑 `install.ps1`

請先在工具資料夾裡執行：

```powershell
.\install.ps1
```

你可以把這一步想成：

**「先確認這份工具包是不是完整可用」**

#### 這一步非常重要：`install.ps1` 不是上網安裝器

它的角色是：

- 檢查這份工具包是否完整
- 檢查能不能直接使用

它不是：

- 上網下載安裝器
- 幫你補抓套件的工具

它內部其實是呼叫：

```powershell
.\setup.ps1 -Offline
```

但你不用記這個技術細節。
你只要記住：

> **`install.ps1` 是檢查，不是下載安裝。**

---

### 4.4 如果 `install.ps1` 失敗，怎麼辦？

請直接照這個原則處理：

- **不要自己研究 npm**
- **不要自己補裝套件**
- **不要自己猜哪個資料夾少了什麼**

請直接聯絡幫你準備工具包的人，請他重新提供完整工具包。

對一般內網使用者來說：

> **`install.ps1` 失敗 = 你拿到的工具包不完整。**

最快的解法通常不是自己修，而是請準備者重做。

---

### 4.5 第三步：啟動專用瀏覽器（預設 Chrome）

檢查通過後，請依照現場要用的瀏覽器執行：

```powershell
# 標準 Chrome 流程
.\launch-chrome.ps1

# 如果現場指定使用 Edge
.\launch-edge.ps1
```

這一步會打開一個**專門給蒐集工具使用的瀏覽器視窗**。

你可以把它想成：

- 這不是在改你平常日常使用的 Chrome / Edge
- 它會用自己的工作資料夾
- 工具會透過這個瀏覽器讀取你目前看到的頁面

#### 執行後你要做什麼？

1. 在這個新開的瀏覽器裡登入你的內部網站
2. 打開你要蒐集的頁面
3. 保持這個瀏覽器開著

---

### 4.6 第四步：開始蒐集

登入完成後，另外開一個 PowerShell 視窗，在同一個工具資料夾裡執行：

```powershell
# 如果前一步用 Chrome
.\collect.ps1

# 如果前一步用 Edge
.\collect.ps1 --browser edge
```

然後照畫面提示操作即可。

如果你是第一次使用，**建議先走預設的互動流程**。

如果你只是想先確認工具有沒有正常運作，也可以用這個最簡單的驗證方式：

```powershell
.\collect.ps1 --snapshot
```

這可以理解成：

**「先快速抓目前這一頁，確認工具有沒有正常工作」**

---

### 4.7 你暫時不用碰哪些東西？

如果你是一般內網使用者，先不用管：

- `setup.ps1`
- `npm run setup`
- `scripts/setup.sh`
- `scripts/prepare-offline-bundle.ps1`
- `node_modules`
- `.playwright-browsers`
- 各種版本檢查指令

只要記住這組入口就好：

- `install.ps1`
- `launch-chrome.ps1`（標準 Chrome）
- `launch-edge.ps1`（指定 Edge 時使用）
- `collect.ps1`（若使用 Edge，請加 `--browser edge`）

---

## 5. 如果失敗怎麼辦？

這一節只講最常遇到、最容易判斷的情況。

### 情況 1：`install.ps1` 失敗

這通常表示：

- 工具包不完整
- 少了必要內容
- 或你拿到的不是完整離線包

#### 你應該怎麼做？

**不要自己研究技術細節。**
直接找準備者重做完整工具包。

---

### 情況 2：啟動瀏覽器入口失敗

你可以先檢查：

- 如果你跑的是 `launch-chrome.ps1`，這台電腦有沒有安裝 Google Chrome
- 如果你跑的是 `launch-edge.ps1`，這台電腦有沒有安裝 Microsoft Edge
- 是不是有別的同類瀏覽器視窗還沒關掉
- 重新執行一次有沒有恢復正常

如果還是不行，請把 `logs\` 裡最新的啟動日誌交給準備者或技術人員。

---

### 情況 3：`collect.ps1` 啟動了，但蒐集不起來

你可以先確認：

- 你是不是用對應的 `launch-chrome.ps1` / `launch-edge.ps1` 開的那個瀏覽器
- 如果你開的是 Edge，`collect.ps1` 有沒有加上 `--browser edge`
- 你有沒有在那個瀏覽器裡登入網站
- 目標頁面是不是已經打開
- 你是不是在同一份工具資料夾裡執行 `collect.ps1`

如果還是不行，也請把 `logs\` 裡最新的蒐集日誌提供出去。

---

### 情況 4：我看到很多技術字眼，會不會要自己修？

一般情況下，**不用**。

請記住這個判斷方式：

- 你是**準備者**：才需要處理打包與補齊環境
- 你是**內網使用者**：以使用為主，不需要自己研究安裝細節

---

## 6. 蒐集完成後，去哪裡找結果？

每次蒐集完成後，結果通常會放在：

```text
materials\
```

裡面的某個新資料夾，通常長得像這樣：

```text
materials\時間戳記_任務名稱\
```

常見內容包括：

```text
materials\時間戳記_任務名稱\
├─ aria-snapshots\
├─ screenshots\
├─ recordings\
├─ metadata.json
└─ summary-report.md
```

### 這些東西白話來說是什麼？

- `aria-snapshots\`
  - 頁面結構紀錄
  - 之後交給 AI 時通常很重要

- `screenshots\`
  - 畫面截圖

- `recordings\`
  - 你的操作錄製

- `metadata.json`
  - 這次任務的基本資訊

- `summary-report.md`
  - 簡短整理報告

如果過程中有錯誤，請再看：

```text
logs\
```

裡最新的 `.log` 檔案。

## 7. 其他補充：真的需要時再看這裡

這一段不是主流程，但需要時很好用。

### 7.1 `collect.ps1` 的常見用法

#### 最推薦：直接執行

```powershell
.\collect.ps1
```

這是最適合第一次使用者的方式。

#### 如果你只想快速抓目前這一頁

```powershell
# Chrome
.\collect.ps1 --snapshot

# Edge
.\collect.ps1 --snapshot --browser edge
```

這比較像「先快速拍一張當前頁面資料」。

#### 如果你是固定工作、重複執行同一套流程

之後才考慮自動模式。
第一次使用時，先從一般互動流程開始就好。

---

### 7.2 `setup.ps1` 是做什麼的？我要不要管它？

大多數內網使用者**不用管它**。

#### 白話說明

`setup.ps1` 是比較偏技術人員用的腳本。
它的用途比較像：

- 補齊工具缺少的零件
- 補齊工具需要的瀏覽器元件
- 處理原始專案的準備工作

#### 你只需要記住

- **一般使用者**：先跑 `install.ps1`
- **技術人員 / 準備者**：必要時才用 `setup.ps1`

---

### 7.3 怎麼把素材交給 AI？

如果你要請 AI 幫你產生腳本，建議附上同一次蒐集的這些內容：

- `aria-snapshots\` 裡的檔案
- `screenshots\` 裡的檔案
- `recordings\` 裡的檔案（如果有）
- `metadata.json`

如果你是要請 AI 幫你除錯，再加上：

- 你目前的腳本檔
- `logs\` 裡相關的日誌檔

#### 很重要

**不要把很長的內容整段貼進對話框。**
直接用附件上傳通常比較好。

---

### 7.4 生成腳本範本

先把 **ARIA 快照、截圖、錄製檔（如果有）、metadata.json** 用附件方式上傳，再貼上下面這段：

```text
請根據我剛剛上傳的附件，幫我產生一份可直接執行的 Playwright TypeScript 腳本。我的目標是【請替換：你想自動完成的工作，例如：登入後查詢資料並匯出結果】；成功標準是【請替換：你怎樣算成功，例如：畫面顯示查詢結果並完成下載】；執行環境是【請替換：你的環境，例如：Windows 11、已開啟 Chrome 或 Edge Debug、已登入內部網站】。請優先根據附件中的 ARIA 快照判斷頁面結構，參考錄製檔理解操作順序，用截圖輔助確認畫面，用 metadata.json 補足背景與順序；如果附件不足，請先明確告訴我缺哪個檔案或哪個步驟，不要自行猜測。請輸出：1. 流程摘要，2. 完整 TypeScript 腳本，3. 需要我先準備的環境變數或參數，4. 容易失敗的步驟與替代做法。附件請直接讀取我上傳的 ARIA 快照、錄製檔、截圖、metadata.json，不要要求我把長內容貼到訊息裡。
```

---

### 7.5 除錯範本

先把 **目前腳本、出錯頁面的 ARIA 快照、相關截圖、錄製檔（如果有）、metadata.json、log** 用附件方式上傳，再貼上下面這段：

```text
請幫我找出這份 Playwright TypeScript 腳本失敗的原因，並直接提供可替換的修正版。我的目標是【請替換：原本想完成的工作】；我預期看到的結果是【請替換：成功時應該發生什麼】；實際遇到的問題是【請替換：錯誤訊息或失敗現象】；大約失敗在【請替換：第幾步或哪個頁面】。請先根據我上傳的附件比對腳本、ARIA 快照、錄製檔、截圖、metadata.json 與 log，告訴我最可能的 1 到 3 個原因，再提供完整修正版或清楚標示要替換的程式碼段落，並說明下次如何避免同樣問題；如果附件還不夠，請直接列出還缺哪個檔案，不要自行猜測。附件請直接讀取我上傳的腳本、ARIA 快照、錄製檔、截圖、metadata.json、log，不要要求我把長內容整段貼到訊息裡。
```

---

### 7.6 安全提醒

- **不要上傳密碼、token、`.env` 檔案。**
- 錄製檔上傳前，仍建議快速檢查一次。
- 截圖如果拍到個資或敏感資訊，先確認再帶出內網。
- 優先上傳**同一次任務資料夾**內的內容，不要把不同任務混在一起。

---

### 7.7 名詞白話版：真的看不懂時再看這裡

#### 工具需要的零件資料夾（`node_modules`）
你可以把它想成這個工具運作時要用到的一堆元件。

#### 內建執行環境（`runtime\node`）
你可以把它想成工具自己的小引擎。
如果準備者有把它一起打包，內網使用者通常就不用另外裝對應環境。

#### 內建瀏覽器元件（`.playwright-browsers`）
這是工具需要的 Playwright Chromium runtime。
一般使用者不用自己處理它，只要準備者有打包完整即可。
就算現場用的是 Edge，這個資料夾也不能少；真正的 Microsoft Edge 仍是電腦裡原本安裝好的系統瀏覽器。

#### 離線包
就是已經先在有網路電腦準備好的完整工具資料夾，可以直接複製到內網使用。

#### 問題紀錄檔（log）
如果工具執行失敗，通常會在 `logs\` 裡留下記錄。
這是你回報問題時最有幫助的資料之一。

### 7.8 如果你真的想知道原理，再看這裡

這個工具底層使用 Playwright。
但對大多數人來說，你不需要先懂這個原理才能使用。

只要先記住你的主流程是：

- `install.ps1`
- `launch-chrome.ps1`
- `collect.ps1`

如果你們單位指定使用 Edge，則改成：

- `install.ps1`
- `launch-edge.ps1`
- `collect.ps1 --browser edge`

就夠了。

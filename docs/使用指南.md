## 1. 蒐集素材（3 步驟）

> 🎯 **目標：把你在 Edge 上看到的頁面，保存成一包素材，等一下交給 AI。**

### 步驟 1-1：檢查工具包

打開 PowerShell，進入工具資料夾，執行：

```powershell
.\install.ps1
```

**你會看到：**

```
============================================================
  🔍 RPA-Cowork：檢查離線工具包
============================================================

✅ runtime\node\ ............ 存在
✅ node_modules\ ............ 存在
✅ .playwright-browsers\ .... 存在
✅ .env.example ............. 存在

🎉 工具包完整，可以開始使用。
```

> ✅ 全部通過 → 繼續下一步。
>
> ❌ 出現紅字 → 把 `logs\` 資料夾裡最新的 `.log` 檔交給幫你準備工具包的人，請對方重新打包。不要自己安裝 Node.js 或 npm。

### 步驟 1-2：開啟 Edge 專用視窗

```powershell
.\launch-edge.ps1
```

**這時會跳出一個新的 Edge 視窗。** 在這個視窗裡做兩件事：

1. 登入你要操作的目標網站
2. 打開你要蒐集的那個頁面

> ⚠️ **不要關掉這個 Edge 視窗。** 下一步需要透過它來蒐集素材。

### 步驟 1-3：開始蒐集

```powershell
.\collect.ps1 --browser edge
```

**你會看到選單：**

```
請選擇模式：
  [1] 互動模式（推薦新手）
  [2] 全自動模式
  [3] 快照模式
  [4] 錄製模式
```

**輸入 `1` 然後按 Enter。**

系統接著會問你兩個問題，照這樣填寫：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `📌 請輸入專案名稱：` | `case-portal` | 用英文代號，代表哪個系統 |
| `📌 請輸入本次任務名稱：` | `案件查詢` | 用中文，描述這次要做什麼 |

系統會自動連上你的 Edge 頁面開始蒐集。

### ✅ 蒐集完成

你會在資料夾裡看到這樣的結構：

```text
materials\
└─ 20260318143000_案件查詢\
   ├─ aria-snapshots\       ← 頁面結構（AI 的核心素材）
   ├─ screenshots\          ← 畫面截圖
   ├─ recordings\           ← 操作錄影（選了錄製模式才有）
   ├─ metadata.json         ← 背景資訊
   └─ summary-report.md     ← 摘要報告
```

> **這包素材就是等一下要上傳給 AI 的附件。保持整包不動，不要拆散。**

### 📋 完整操作範例

> **情境：蒐集「案件查詢系統」的查詢頁面。**

| 你輸入的指令 | 你會看到什麼 |
|-------------|------------|
| `.\install.ps1` | ✅ 工具包完整 |
| `.\launch-edge.ps1` | Edge 視窗跳出來 |
| （在 Edge 裡登入案件查詢系統） | 登入成功，看到查詢頁面 |
| `.\collect.ps1 --browser edge` | 出現選單 |
| 輸入 `1` | 選擇互動模式 |
| 輸入 `case-portal` | 設定專案名稱 |
| 輸入 `案件查詢` | 設定任務名稱 |
| （等待蒐集完成） | 素材出現在 `materials\` 資料夾 |

## 2. 請 AI 幫你寫腳本（3 步驟）

> 🎯 **目標：讓 AI 把你蒐集的素材，變成一支可以直接執行的自動化腳本。**

### 步驟 2-1：建立腳本骨架

在 PowerShell 執行：

```powershell
.\new-task.ps1
```

系統會問你兩個問題：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `📌 請輸入系統代碼：` | `case` | 英文，識別哪個系統 |
| `📌 請輸入任務名稱：` | `案件查詢` | 中文，描述要做什麼 |

完成後你會看到：

```
✅ 腳本已建立：src\case-案件查詢.ts
```

> 你不用看懂骨架檔的內容。它的作用是告訴 AI：**「在這個框架裡幫我補完，不要另建新專案。」**

### 步驟 2-2：上傳附件給 AI

打開 [Claude](https://claude.ai)，確認使用 **Claude Sonnet 4.6** 模型，開始新對話。

**點迴紋針圖示 📎，依序上傳以下檔案：**

| # | 上傳什麼 | 到哪裡找 |
|---|---------|---------|
| 1 | 骨架檔 `src\case-案件查詢.ts` | 工具資料夾 → `src\` |
| 2 | 所有 `.txt` 檔 | `materials\20260318143000_案件查詢\aria-snapshots\` |
| 3 | 所有 `.png` 檔 | `materials\20260318143000_案件查詢\screenshots\` |
| 4 | `metadata.json` | `materials\20260318143000_案件查詢\` |
| 5 | 所有 `.js` 檔（錄製模式的產出） | `materials\20260318143000_案件查詢\recordings\` |

> ⚠️ **絕對不要上傳 `.env` 檔案。** `.env` 裡面有你的帳號密碼。

### 步驟 2-3：貼上 Prompt 給 AI

上傳完附件後，**複製下方 Prompt 範本，貼到 Claude 的訊息框送出。**

```
📌 使用方式：
   看到 ⬅ 的那一行 → 一定要改成你自己的內容
   沒有 ⬅ 的那一行 → 原封不動保留，直接送出
```

> 範本裡已經預填了一個「案件查詢」的範例。
> 你只要把有 `⬅` 的行改成你自己的狀況，其他全部保留。

## 3. 執行腳本（2 步驟）

> 🎯 **目標：把 AI 寫好的腳本存回來，然後執行。**

### 步驟 3-1：存回 AI 的成果

1. 在 Claude 的回覆裡，找到完整的程式碼區塊（灰底的區塊）
2. 點程式碼區塊右上角的 **複製** 按鈕
3. 用記事本打開你的腳本檔（例如 `src\case-案件查詢.ts`）
4. `Ctrl+A` 全選 → `Ctrl+V` 貼上 → `Ctrl+S` 存檔

**接著設定 .env（Claude 回覆的第 4 點會告訴你要填什麼）：**

```powershell
Copy-Item .env.example .env
notepad .env
```

在 `.env` 裡填入 Claude 指定的欄位值，例如：

```text
TARGET_URL=https://case.internal/query
CASE_USERNAME=你的帳號
CASE_PASSWORD=你的密碼
```

> ⚠️ `.env` 只放在你自己的電腦上，絕對不要上傳或傳給其他人。
>
> 💡 Claude 回覆第 4 點寫「無」→ 跳過這一步，直接執行。

### 步驟 3-2：執行

確認 Edge 專用視窗還開著，然後執行：

```powershell
.\run-task.ps1 src\case-案件查詢.ts
```

**你會看到：**

```
============================================================
  🤖 RPA-Cowork Task Runner
============================================================

📋 任務名稱：案件查詢
📂 專案根目錄：C:\Users\你的名字\RPA-Cowork
🕐 執行時間：2026-03-18T20:30:00+08:00

🌐 正在開啟目標頁面：https://case.internal/query
🔐 等待登入完成...
✅ 案件查詢 任務完成

============================================================
  ✅ 任務執行成功
============================================================
```

> 🎉 看到 **✅ 任務執行成功** 就完成了！

### 執行失敗的處理流程

**不要自己改程式碼。** 依序做以下 3 件事，然後用 [Prompt B](#-prompt-b請-ai-修正失敗的腳本) 請 AI 修正：

| 步驟 | 做什麼 | 怎麼做 |
|------|-------|--------|
| 1 | 複製錯誤訊息 | 在 PowerShell 視窗裡用滑鼠選取紅色文字 → 按右鍵複製 |
| 2 | 找日誌檔 | 到 `logs\` 資料夾，找最新的 `.log` 檔 |
| 3 | 補最新快照 | 執行 `.\collect.ps1 --snapshot --browser edge` |

把這 3 樣東西和原本的附件一起上傳給 Claude，用 [Prompt B](#-prompt-b請-ai-修正失敗的腳本) 貼上修正請求。

## 4. 遇到問題怎麼辦

> **按照你看到的症狀，直接找對應的解法。**

---

### ❌ `install.ps1` 顯示紅色錯誤

**原因：** 離線包不完整。

**解法：** 把 `logs\` 最新的 `.log` 交給幫你準備工具包的人，請對方重新打包。不要自己安裝 Node.js 或 npm。

---

### ❌ `launch-edge.ps1` 沒有反應

**排查：**

1. 確認這台電腦有安裝 Microsoft Edge
2. 確認你在完整的工具資料夾裡執行（不是只有 `.ps1` 檔案）
3. 重新執行一次 `.\launch-edge.ps1`

還是不行 → 把 `logs\` 最新日誌交給準備工具包的人。

---

### ❌ `collect.ps1 --browser edge` 顯示「找不到頁面」

**排查：**

1. 確認你是用 `.\launch-edge.ps1` 開出來的那個 Edge 視窗（不是自己另外開的）
2. 確認那個 Edge 視窗還開著
3. 確認你已經在 Edge 裡登入了目標網站
4. 確認指令有加 `--browser edge`

---

### ❌ AI 叫你 `npm install` 或 `npx`

**原因：** AI 不知道專案已有框架，以為你要從零開始。

**解法：** 用 [Prompt C](#-prompt-cai-回覆不對叫你-npm-install-或另建專案) 補上框架檔案再問一次。

---

### ❌ `run-task.ps1` 顯示「缺少 .env」

**解法：**

```powershell
Copy-Item .env.example .env
notepad .env
```

填入 Claude 說需要的欄位值，存檔後重新執行。

---

### ❌ 腳本跑到一半停住或紅色錯誤

**解法：** 按照 [執行失敗的處理流程](#執行失敗的處理流程) 蒐集錯誤資訊，用 [Prompt B](#-prompt-b請-ai-修正失敗的腳本) 請 AI 修正。

---

### ❌ PowerShell 顯示「已停用指令碼執行」

**解法：** 用系統管理員身分打開 PowerShell，執行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

單位有更嚴格的政策 → 請依內部規範處理。

---

### 💡 使用 Chrome

本指南主線是 Edge。使用 Chrome 時替換這兩個指令，其他步驟完全相同：

| Edge 指令 | Chrome 指令 |
|----------|------------|
| `.\launch-edge.ps1` | `.\launch-chrome.ps1` |
| `.\collect.ps1 --browser edge` | `.\collect.ps1` |

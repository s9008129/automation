## 0. 前言：為什麼選 Playwright，不選 Selenium？

> 你不需要懂這兩個工具的技術細節。
> 下面用三個比喻說明，看完你就知道為什麼這個專案做了這個選擇。

### 🎯 比喻一：找按鈕的方式不同

**Selenium 的做法**像是有人告訴你：「往右走 300 步、往下走 200 步，那裡有一個按鈕，按下去。」

問題是：辦公室重新裝潢過（網頁改版），或你的螢幕比較小（解析度不同），那個按鈕就不在那個位置了。你照著走，結果按到空氣。

**Playwright 的做法**像是直接告訴你：「找到寫著『查詢』兩個字的按鈕，按下去。」

不管辦公室怎麼改裝潢、你的螢幕多大多小，只要那個按鈕還在頁面上，它就找得到。

### 🔄 比喻二：網頁改版時的抗震能力

內網系統常常改版——換個版型、調個選單位置、多一欄少一欄。

用座標和固定路徑去找元素的方式，就像把家具位置全部背下來，然後閉著眼走路。**家具一搬，就撞牆。**

Playwright 用的是頁面上的「文字」和「語意結構」（例如：這是一個按鈕、這是一個輸入欄位）來找目標。就像你走進一間重新裝潢的辦公室，不用背家具位置，直接看門牌找房間就好。**家具搬了，門牌還在。**

### 🔧 比喻三：少一個要維護的東西

Selenium 需要一個叫「driver」的東西，作用像是瀏覽器的鑰匙。**瀏覽器更新了，鑰匙也要換。** 版本沒對上，工具就打不開瀏覽器。

在內網環境，你不一定能隨時上網下載新的 driver。結果就是：瀏覽器自動更新了，driver 沒跟上，整個工具就不能用了。

Playwright 不需要這種 driver 對版。**鑰匙跟鎖是一體的，不用自己配。** 這對離線環境來說省掉很多麻煩。

### 📋 三句話總結

| 痛點 | Selenium | Playwright |
|------|----------|------------|
| 找按鈕的方式 | 靠座標走路，位置一變就按到空氣 | 靠文字和語意找，位置變了照樣找到 |
| 網頁改版 | 路徑全壞，要重新調整 | 只要元素文字還在，通常不用改 |
| 瀏覽器更新 | driver 要配對版本，沒對上就不能用 | 不需要額外 driver，離線包自帶 |

> **所以 RPA-Cowork 選擇 Playwright：更穩定、更省事、更適合離線環境。**

## 1. 蒐集素材（3 步驟）

> 🎯 **目標：把你在瀏覽器裡看到的重要頁面，保存成一包素材，等一下交給 AI。**

### 步驟 0：打開 PowerShell 進入工具資料夾

> 🎯 **每次操作都從這裡開始。** 不管是蒐集素材、建骨架、還是執行腳本，第一步都是打開 PowerShell 並進入工具資料夾。

1. 在你電腦上找到工具資料夾（技術準備者交給你的那個資料夾，例如 `D:\RPA-Cowork` 或桌面上的 `RPA-Cowork`）
2. 打開該資料夾，在**空白處**按住 `Shift` 再按**滑鼠右鍵**
3. 選擇「**在這裡開啟 PowerShell 視窗**」（或「在終端中開啟」）

**如果右鍵選單沒有這個選項，改用這個方式：**

1. 按鍵盤 `Win + R`，輸入 `powershell`，按 Enter
2. 在 PowerShell 裡輸入以下指令切換到工具資料夾：

```powershell
cd "D:\RPA-Cowork"
```

> ⚠️ 把 `D:\RPA-Cowork` 換成你自己工具資料夾的實際路徑。

**確認你在對的位置：** 輸入 `dir install.ps1` 然後按 Enter。如果有看到檔案資訊就對了。如果顯示「找不到」，表示路徑不對，請重新確認資料夾位置。

在 PowerShell 裡執行（如果還沒打開 PowerShell，先回到[步驟 0](#步驟-0打開-powershell-進入工具資料夾)）：

```powershell
.\install.ps1
```

```
  ✅ install / collect / launch（Chrome / Edge）與 .env.example 已就緒
```

> ✅ 全部通過 → 繼續下一步。

> ❌ 出現紅字 → 把 `logs\` 資料夾裡最新的 `.log` 檔交給幫你準備工具包的人，請對方重新打包。不要自己安裝 Node.js 或 npm。

### 步驟 1-2：開啟 Edge 專用視窗

```powershell
.\launch-edge.ps1
```

**這時會跳出一個新的 Edge 視窗。** 在這個視窗裡做兩件事：

1. 登入你要操作的目標網站
2. 打開你要蒐集的那個頁面

> ⚠️ **不要關掉這個 Edge 視窗。** 下一步需要透過它來蒐集素材。
>
> ⚠️ **這個 Edge 視窗是「專用視窗」，和你平常用的 Edge 瀏覽器是分開的。** 蒐集指令只認這個專用視窗裡的頁面。如果你在自己平常的 Edge 登入目標網站，蒐集指令會抓不到。
>
> 💡 **SSO / 單一登入系統注意：** 很多內網系統（例如 QIZ、EIP）登入時會跳出新視窗或彈出框做身分驗證（SSO 流程）。這是正常的——在專用視窗裡完成整個登入流程（包含跳出的驗證視窗），確認看到目標頁面後再繼續下一步。

### 步驟 1-3：開始蒐集

```powershell
.\collect.ps1 --browser edge
```

**你會看到選單：**

```
  請選擇蒐集模式：

  [1] 📸 互動模式（推薦新手）- 一步一步引導你蒐集
  [2] 🤖 自動模式 - 依設定檔自動蒐集所有頁面
  [3] ⚡ 快照模式 - 快速擷取當前頁面
  [4] 🎬 錄製模式 - 啟動 Codegen 錄製互動流程
```

**輸入 `1` 然後按 Enter。**

系統接著會問你三個問題，照這樣填寫：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `專案名稱:` | `tax-cloud` | 用英文代號，代表哪個系統 |
| `素材根目錄 (Enter=./materials):` | 直接按 Enter | 預設存到 `materials\` 資料夾 |
| `本次任務/錄製名稱（Enter 使用「tax-cloud」）:` | `批次簽核` | 描述這次要做什麼 |

系統會自動連上你的 Edge 頁面，然後進入互動模式。

### 🔄 互動模式完整流程

互動模式分成 **兩個階段**，你可以隨時在兩個階段之間來回切換：

```
階段 1/2：頁面蒐集          階段 2/2：流程錄製
┌─────────────────┐        ┌─────────────────┐
│ 擷取 ARIA 快照   │  ←→    │ 錄下你的操作流程  │
│ 擷取截圖         │        │ （會開新視窗）    │
└─────────────────┘        └─────────────────┘
        ↓                          ↓
   蒐集完所有頁面後          錄製完所有流程後
        ↓                          ↓
        └──────── 結束並產生報告 ────┘
```

進入互動模式後，你會先看到：

```
  ────────────────────────────────────────────────────────────
  階段 1/2：頁面蒐集（ARIA-first）
  請先到已登入的 Edge，把畫面切到你要蒐集的頁面，再回來依選單操作。
  每蒐集完一頁，你都可以直接切到錄製階段，不需要中斷程式。
  ────────────────────────────────────────────────────────────
```

接著出現選單：

```
  頁面蒐集階段：接下來要做什麼？
    [1] 蒐集目前 Edge 頁面（第 1 頁） — 先把 Edge 切到正確頁面
    [2] 切換到錄製階段 — 改為錄下互動流程
    [3] 結束並產生報告 — 本次蒐集先到這裡
```

**輸入 `1`** → 系統會偵測你 Edge 目前的頁面，顯示：

```
  目前準備蒐集第 1 頁

  請先確認 Edge 現在顯示的就是你要蒐集的頁面，再輸入下面資訊。
```

然後問你兩個問題：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `頁面名稱（例如：01-login-page）:` | `01-login-page` | 英文編號，方便辨識 |
| `頁面說明（Enter 使用「財稅雲端平台入口網」）:` | 直接按 Enter | 系統會用頁面標題當說明 |

蒐集完成後你會看到：

```
  ✅ 這一頁的素材已完成
    - 頁面名稱：01-login-page
    - 頁面標題：財稅雲端平台入口網
    - ARIA 快照：aria-snapshots/01-login-page.txt
    - 截圖：screenshots/01-login-page.png
```

接著出現下一步選單：

```
  本頁完成，下一步要做什麼？
    [1] 繼續蒐集下一個頁面 — 回到 Edge 切換到下一頁後再操作
    [2] 進入錄製階段 — 開始錄下操作流程
```

> 💡 **要蒐集多個頁面：** 在 Edge 裡切換到下一個頁面 → 回到 PowerShell 選 `1` → 重複以上步驟。
> 每蒐集一頁，系統會自動把 ARIA 快照和截圖存好。

#### 階段 2：流程錄製

選擇「切換到錄製階段」後，你會看到：

```
  ────────────────────────────────────────────────────────────
  階段 2/2：流程錄製
  這個階段會開啟一個新的 Playwright 錄製視窗，和你原本的 Edge 分開。
  請在新視窗完成操作，錄製結束時直接關閉該視窗即可。
  視窗關閉後，請回到這個 PowerShell，依照選單決定下一步。
  ────────────────────────────────────────────────────────────
```

選單：

```
  錄製階段：接下來要做什麼？
    [1] 開始新的錄製 — 會開新的 Playwright 錄製視窗
    [2] 返回頁面蒐集階段 — 繼續擷取 ARIA 快照與截圖
    [3] 結束並產生報告 — 儲存本次成果
```

**輸入 `1`** → 系統會問你錄製資訊：

| 系統問你 | 你輸入 | 說明 |
|---------|-------|------|
| `錄製名稱（Enter 使用「01-login-page-flow」）:` | 直接按 Enter | 系統會自動命名 |
| `起始 URL（Enter 使用「https://...」）:` | 直接按 Enter | 系統會用你剛才蒐集的頁面 URL |
| `操作說明（可選，例：登入後查詢案件並送出）:` | `登入→進入批次簽核→全選→F6處理` | 描述你要錄下什麼操作 |

接著一個 **新的瀏覽器視窗** 會跳出來。在這個新視窗裡操作你要錄製的流程：

1. 在新視窗裡完成你的操作（例如：登入 → 進入批次簽核 → 全選 → F6 批次處理）
2. 操作完畢後，**直接關閉新視窗**
3. 回到 PowerShell，系統會自動保存錄製檔

```
  ✅ 錄製檔已納入本次成果
    - 流程名稱：01-batch-flow
    - 錄製檔：recordings/01-batch-flow.ts
    - 目前累計：1 頁面 / 1 錄製 / 0 錯誤
```

> ⚠️ **錄製視窗和你原本的 Edge 是兩個完全獨立的環境。**
> - 錄製視窗有自己的登入狀態——你在裡面登入的帳號，和你原本 Edge 專用視窗裡的登入互不影響。
> - 關掉錄製視窗不會影響你已登入的 Edge 專用視窗。
> - 如果你的系統登入時會跳出 SSO 彈出視窗（例如 QIZ 的單一登入），在錄製視窗裡也會遇到同樣的流程，這是正常的。

不管在哪個階段，選 **「結束並產生報告」** 就會結束整個流程，產生摘要報告。

### 💡 為什麼這些素材這麼重要？

> 以下說明 AI 如何利用你蒐集的每種素材來產出正確的腳本。

| 你蒐集的素材 | AI 從裡面看到什麼 | 為什麼重要 |
|------------|------------------|-----------|
| ARIA 快照（`.txt`） | 頁面結構：哪些按鈕叫什麼名字、表單有幾個欄位、iframe 層級關係 | AI 靠這個寫出正確的「找按鈕」指令，不會按到空氣 |
| 截圖（`.png`） | 畫面實際長相：按鈕是否可以按、有沒有分頁、目前顯示幾筆資料 | AI 靠這個確認邏輯：例如「看到共 66 筆 / 2 頁，所以要寫迴圈處理所有頁」 |
| 錄製檔（`.ts`） | 你的操作順序：先點哪裡、再填什麼、最後按哪個按鈕 | AI 靠這個理解流程，但會**主動找出錯誤**並修正（例如找出硬編碼的網址、漏掉的分頁處理） |
| `metadata.json` | 蒐集環境：用的什麼瀏覽器、什麼系統、什麼時間 | AI 靠這個決定腳本要用 Edge 還是 Chrome、時區設定等 |

> 📌 **重點：素材蒐集得越完整，AI 產出的腳本品質越高。** 不要只蒐集一個頁面就急著給 AI——把流程中會經過的每個重要頁面都蒐集下來。

### 執行失敗的處理流程

**不要自己改程式碼。** 依序做以下步驟蒐集資料，然後用 [Prompt B](#-prompt-b請-ai-修正失敗的腳本) 請 AI 修正：

| 步驟 | 做什麼 | 怎麼做 |
|------|-------|--------|
| 1 | **不要關掉 Edge** | 腳本跑到一半失敗時，Edge 可能停在出錯的頁面。先留著不動 |
| 2 | 複製錯誤訊息 | 在 PowerShell 視窗裡用滑鼠選取紅色文字 → 按右鍵複製，存到一個文字檔備用 |
| 3 | 找日誌檔 | 到 `logs\` 資料夾，找日期最新的 `.log` 檔（檔名包含日期，找數字最大的） |
| 4 | 補最新快照 | Edge 停在出錯頁面的狀態下，執行 `.\collect.ps1 --snapshot --browser edge`，蒐集「失敗當下」的頁面狀態 |
| 5 | 手動截圖（選做） | 用鍵盤 `Print Screen` 截下目前 Edge 畫面，存成 `.png` 圖片 |
| 6 | 檢查 error_log（選做） | 看看工具資料夾根目錄有沒有 `error_log.txt`，有的話一併上傳 |

把以上蒐集到的東西，**連同你的 `.ts` 腳本檔**，一起上傳給 Claude，用 [Prompt B](#-prompt-b請-ai-修正失敗的腳本) 貼上修正請求。完整的附件清單請看 [Prompt B 的附件表格](#-prompt-b請-ai-修正失敗的腳本)。

## 4. 遇到問題怎麼辦

> **按照你看到的症狀，直接找對應的解法。**

---

### ❌ `install.ps1` 顯示紅色錯誤

**原因：** 離線工具包不完整。

**你不需要自己安裝任何東西。** 這不是你的問題。

**解法：**

1. 到 `logs\` 資料夾找最新的 `.log` 檔，傳給你的**技術準備者**（幫你打包工具的人）
2. 請對方檢查並重新打包
3. 拿到新的工具包後，**整個資料夾換掉**，重新執行 `.\install.ps1` 確認全部綠色通過

> ⚠️ **離線工具包不會自動更新。** 如果技術準備者更新了 repo，你手上的工具包也需要重新交付才會生效。請務必拿到重新打包的版本。

---

### ❌ `launch-edge.ps1` 沒有反應

**排查：**

1. 確認這台電腦有安裝 Microsoft Edge
2. 確認你在完整的工具資料夾裡執行（不是只有 `.ps1` 檔案）
3. 重新執行一次 `.\launch-edge.ps1`

還是不行 → 把 `logs\` 最新日誌交給準備工具包的人。

---

### ❌ `collect.ps1 --browser edge` 顯示「找不到頁面」

**排查：**

1. 確認你是用 `.\launch-edge.ps1` 開出來的那個 Edge 視窗（不是自己另外開的）
2. 確認那個 Edge 視窗還開著
3. 確認你已經在 Edge 裡登入了目標網站
4. 確認指令有加 `--browser edge`

---

### ❌ AI 叫你 `npm install` 或 `npx`

**原因：** AI 不知道專案已有框架，以為你要從零開始。

**解法：** 用 [Prompt C](#-prompt-cai-回覆不對叫你-npm-install-或另建專案) 補上框架檔案再問一次。

---

### ❌ `run-task.ps1` 顯示「缺少 .env」

**解法：**

```powershell
Copy-Item .env.example .env
notepad .env
```

填入 Claude 說需要的欄位值，存檔後重新執行。

---

### ❌ 腳本跑到一半停住或紅色錯誤

**解法：** 按照 [執行失敗的處理流程](#執行失敗的處理流程) 蒐集錯誤資訊，用 [Prompt B](#-prompt-b請-ai-修正失敗的腳本) 請 AI 修正。

---

### ❌ 腳本在登入後就停住（SSO 跳出新視窗或頁面跳轉）

**你看到的症狀：** 腳本開啟了登入頁面，你完成帳密輸入或驗證後，畫面跳轉了但腳本不動了。PowerShell 可能顯示類似這樣的錯誤：

```
Timeout 30000ms exceeded waiting for navigation
```

或是停在「等待 SSO 登入完成」之後就沒有下一步。

**原因：** 很多內網系統（例如 QIZ201 批次簽核所在的 EIP 平台）使用 SSO 單一登入。登入過程中系統會跳出新視窗或切換頁面做身分驗證，完成後再跳回主頁面。如果腳本沒有處理這種頁面切換，就會卡住。

**這是常見的系統架構模式，不是你個人操作錯誤。**

**解法：**

1. 用 [Prompt B](#-prompt-b請-ai-修正失敗的腳本) 請 AI 修正
2. 在 Prompt 的「這支腳本目前的問題」裡特別寫上：
   `登入流程會跳出新視窗做 SSO 驗證，完成後才回到主頁面，請處理這個頁面切換`
3. 附上腳本停住時的 ARIA 快照和截圖（在 Edge 停住的畫面上重新執行 `.\collect.ps1 --snapshot --browser edge` 即可）

---

### ❌ 腳本有時能跑、有時跑不起來

**原因：** 離線工具包可能不完整，或內部的 runtime 設定有問題。

**解法：** 請你的**技術準備者**用最新的 repo 重新打包離線工具包。拿到新包後把整個工具資料夾換掉，不要只替換單一檔案。

---

### ❌ 不確定哪個是「最新的」log 或 materials 資料夾

**判斷方式：**

- **`logs\` 資料夾**：檔案名稱包含日期時間，例如 `2026-03-18_203000.log`。**日期數字最大的就是最新的。**
- **`materials\` 資料夾**：子資料夾名稱以日期時間開頭，例如 `20260318203500_批次簽核\`。**開頭數字最大的就是最新的。**

> 💡 在 PowerShell 裡也可以用 `dir logs\ | Sort-Object LastWriteTime -Descending | Select-Object -First 1` 快速找到最新的 log 檔。不過直接用檔案總管看也完全沒問題。

---

### ❌ PowerShell 顯示「已停用指令碼執行」

**解法：** 用系統管理員身分打開 PowerShell，執行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

單位有更嚴格的政策 → 請依內部規範處理。

# 內部網路網頁素材蒐集工具使用指南

> 這份手冊是給第一次使用的人。你可以把這個工具想成「幫網站拍照、畫地圖、記操作」，再把這些附件交給 AI，請它幫你寫自動化腳本。

## 1. 為什麼這次改用 Playwright？

如果你原本比較熟 Selenium，可以先把兩者想成兩種找店方式：

- **Selenium**：比較像「拿地址找店」。你知道店大概在哪裡，但只要門口改一下、畫面換一下，有時就要重新確認。
- **Playwright**：比較像「看招牌找店」。它比較容易直接抓到真正要按的按鈕、欄位或文字，所以面對現在常見的網站，通常更穩一點。

這不是說 Selenium 不好。Selenium 很像大家用很久的老工具：成熟、資料多、很多同事也熟。  
但這次我們要做的是 **蒐集網站素材，再交給 AI 產生腳本**。如果說 Selenium 比較像要自己翻出好幾樣工具，Playwright 就比較像一個已經把「錄製、截圖、看畫面結構」放在一起的工具包，所以這條流程通常更順。

| 工具 | 好處 | 可能要留意 |
|---|---|---|
| Selenium | 大家熟、範例多、很多舊系統也找得到做法 | 畫面一改、等待時機一變，有時就要多調整 |
| Playwright | 對現代網站通常比較穩，也更適合錄製、截圖、蒐集快照 | 需要習慣新的做法，舊 Selenium 範例不能直接整段搬過來 |

**一句話記住：** Selenium 像老工具，Playwright 像新工具箱。這次不是要否定 Selenium，而是選一個更適合目前任務的工具。

## 2. 安裝方法

大多數情況下，這一步你可以自己先完成：平常照步驟跑 `.\setup.ps1` 就好，只有遇到非預期錯誤再找技術人員。

1. **先確認有 PowerShell 與 Node.js**
   - 請用 **PowerShell 7** 開啟這個專案資料夾。
   - 輸入 `node --version`，有看到版本號即可（建議 v20 以上）。
2. **執行安裝腳本**

```powershell
.\setup.ps1
```

3. **等它跑完**
   - 正常情況下，照畫面完成即可。
   - 這一步平常可以自己做，不用每次都找技術人員。
4. **什麼時候再找技術人員**
   - 出現非預期錯誤
   - 安裝失敗
   - 公司電腦跳出權限限制
   - 明明照步驟做，還是無法完成

如果公司同事已經先幫你安裝好，也可以直接跳到下一節。

## 3. 這個工具是做什麼的？

這個工具用來蒐集內部網站的素材，方便之後交給 AI 產生 Playwright 自動化腳本。

- **ARIA 快照**：像網站的「骨架圖」，讓 AI 知道頁面上有哪些按鈕、欄位、表格。
- **截圖**：像「畫面照片」，讓你和 AI 確認畫面長相。
- **錄製檔**：像「操作示範」，讓 AI 知道你先按哪裡、後做什麼。
- **metadata.json**：像「說明卡」，記錄這次蒐集的基本資訊。
- **log**：像「看診紀錄」，出錯時可以交給 AI 協助排查。

## 4. 最短上手流程

如果你剛完成上一節安裝，或公司同事已經幫你裝好工具，就直接從這裡開始。

1. 啟動專用 Chrome：

```powershell
npm run start:chrome
```

2. 在開啟的 Chrome 內登入你的內部網站。
3. 另外開一個 PowerShell 視窗，開始蒐集：

```powershell
npm run collect
```

4. 依照畫面提示選擇頁面、輸入名稱，工具會自動幫你存檔。
5. 如果你還想把「操作流程」也示範給 AI，可以依照畫面提示進入錄製，或另外執行：

```powershell
npm run collect:record
```

> 新手建議：先用 `npm run collect`。這是最容易成功的方式。

## 5. 蒐集完成後，去哪裡看結果？

每次蒐集都會建立一個新的任務資料夾，位置在：

```text
materials\<時間戳記_任務名稱>\
```

常見內容如下：

```text
materials\<時間戳記_任務名稱>\
├─ aria-snapshots\   ← 最重要，AI 主要靠這個理解頁面
├─ screenshots\      ← 畫面截圖
├─ recordings\       ← 操作錄製檔
├─ metadata.json      ← 本次蒐集資訊
└─ summary-report.md  ← 摘要說明
```

如果執行過程有問題，請再看：

```text
logs\
```

裡面最新的 `.log` 檔。

## 6. 什麼情況要蒐集哪些素材？

- **只想讓 AI 看懂頁面**：至少蒐集 **ARIA 快照 + 截圖**。
- **想讓 AI 更容易照著你的步驟做**：再加上 **錄製檔**。
- **想讓 AI 更快理解整體背景**：一起附上 **metadata.json**。
- **腳本失敗、要請 AI 幫你除錯**：再附上 **log**。

不知道怎麼選時，最簡單的做法就是：**把同一次任務資料夾中的檔案，用附件方式一起上傳給 AI。**

## 7. 怎麼把素材交給 AI

1. 找到同一次蒐集的任務資料夾。
2. 在 AI 對話中，用 **附件上傳** 這些檔案：
   - `aria-snapshots\` 裡的檔案
   - `screenshots\` 裡的檔案
   - `recordings\` 裡的檔案（如果有）
   - `metadata.json`
3. 如果是除錯，再加上：
   - 你目前的腳本檔
   - `logs\` 裡相關的 `.log`
4. **不要把長篇 ARIA、錄製檔、metadata 或 log 內容整段貼進訊息裡。** 直接上傳附件就好。

## 8. 生成腳本範本

先把 **ARIA 快照、截圖、錄製檔（如果有）、metadata.json** 用附件方式上傳，再貼上下面這一段：

```text
請根據我剛剛上傳的附件，幫我產生一份可直接執行的 Playwright TypeScript 腳本。我的目標是【請替換：你想自動完成的工作，例如：登入後查詢資料並匯出結果】；成功標準是【請替換：你怎樣算成功，例如：畫面顯示查詢結果並完成下載】；執行環境是【請替換：你的環境，例如：Windows 11、已開啟 Chrome Debug、已登入內部網站】。請優先根據附件中的 ARIA 快照判斷頁面結構，參考錄製檔理解操作順序，用截圖輔助確認畫面，用 metadata.json 補足背景與順序；如果附件不足，請先明確告訴我缺哪個檔案或哪個步驟，不要自行猜測。請輸出：1. 流程摘要，2. 完整 TypeScript 腳本，3. 需要我先準備的環境變數或參數，4. 容易失敗的步驟與替代做法。附件請直接讀取我上傳的 ARIA 快照、錄製檔、截圖、metadata.json，不要要求我把長內容貼到訊息裡。
```

## 9. 除錯範本

先把 **目前腳本、出錯頁面的 ARIA 快照、相關截圖、錄製檔（如果有）、metadata.json、log** 用附件方式上傳，再貼上下面這一段：

```text
請幫我找出這份 Playwright TypeScript 腳本失敗的原因，並直接提供可替換的修正版。我的目標是【請替換：原本想完成的工作】；我預期看到的結果是【請替換：成功時應該發生什麼】；實際遇到的問題是【請替換：錯誤訊息或失敗現象】；大約失敗在【請替換：第幾步或哪個頁面】。請先根據我上傳的附件比對腳本、ARIA 快照、錄製檔、截圖、metadata.json 與 log，告訴我最可能的 1 到 3 個原因，再提供完整修正版或清楚標示要替換的程式碼段落，並說明下次如何避免同樣問題；如果附件還不夠，請直接列出還缺哪個檔案，不要自行猜測。附件請直接讀取我上傳的腳本、ARIA 快照、錄製檔、截圖、metadata.json、log，不要要求我把長內容整段貼到訊息裡。
```

## 10. 安全提醒

- **不要上傳密碼、token、`.env` 檔案。**
- 錄製檔雖然通常會自動清掉密碼，但 **上傳前還是請快速看一眼**。
- 截圖可能拍到個資或敏感資訊，帶出內網前請先確認。
- 不確定要給 AI 哪些檔案時，優先交 **同一次任務資料夾** 的檔案，不要把不同任務混在一起。

## 11. 常見問題 / 快速排除

| 問題 | 你可以先做什麼 |
|---|---|
| Chrome 連不上 | 重新執行 `npm run start:chrome`，並確認你登入的是那個新開的 Chrome |
| 找不到要蒐集的頁面 | 先把目標網站頁面切到前景，再執行 `npm run collect` |
| 只想快速抓目前這一頁 | 用 `npm run collect:snapshot` |
| 錄製完沒有看到檔案 | 再試一次錄製；如果還是不行，先把已有素材交給 AI，再附上 log 請它幫你判斷 |
| AI 說資料不夠 | 補上同一次任務的 ARIA 快照、截圖、錄製檔、metadata.json |
| 腳本執行失敗 | 把腳本、出錯頁面的 ARIA 快照、截圖、相關 log 一起用附件上傳，套用上面的除錯範本 |

## 12. 一句話記住

**先登入、再蒐集、看結果、用附件上傳給 AI。**  
如果想更穩，記得：**ARIA 快照最重要，錄製檔是加分，log 是除錯用。**

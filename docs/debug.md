# Prompt B：請 AI 修正失敗的腳本

## 🟡 使用時機

當腳本執行後出現錯誤（例如 Playwright 找不到元素、逾時、登入失敗等），使用此 Prompt 請 AI 協助 debug 並修正。

## 📋 Prompt 內容

請將以下內容複製後貼給 AI，並把 `【】` 內的部分替換成你實際的錯誤訊息與腳本內容：

---

```
你是一位熟悉 Playwright + TypeScript 的 RPA 開發者。
以下腳本執行時發生錯誤，請幫我找出原因並提供修正版本。

## 錯誤訊息

【請貼上完整的錯誤輸出，例如：
  TimeoutError: page.click: Timeout 30000ms exceeded.
  ...
】

## 目前腳本內容

【請貼上完整的 .ts 腳本內容】

## 修正要求

- 請保持原有的專案框架結構（runTaskEntry、lib/* 匯入等）不變
- 若需要調整 selector，請說明你的判斷依據
- 若問題是頁面載入時機，請加入適當的 waitFor 或重試機制
- 修正後的腳本請直接輸出完整內容，方便我直接替換

## 補充資訊（選填）

【如有以下資訊請一併提供，有助於更精準的修正：
  - 執行環境：Windows 11 / Edge 版本
  - 目標網站截圖或 HTML 片段
  - 上次成功執行的版本（如有）
】
```

---

## 📌 注意事項

- 執行腳本的指令：`.\run-task.ps1 src\你的腳本.ts`
- 若問題重複發生，建議將穩定的 selector 或等待邏輯抽入 `src/lib/` 共享模組
- 修正完成後，建議回到 [生成腳本.md](生成腳本.md) 確認腳本是否符合開發規範
